# 时间画像与结果学习：实现说明

## 目标

电子军师需要记住过去，但不能把过去冻结成永久人格。v4 的目标是：

- 原始观察不可被悄悄覆盖；
- 新行为可以逐渐改变当前判断；
- 少量样本不能让权重剧烈摆动；
- 用户能看懂为什么权重变化；
- 实际行动始终能与聊天甜度分开。

## 事件溯源

每次反馈先写入 `feedback_events`。对画像有意义的信号再追加到 `trait_observations`：

```text
profile_slug + trait_key + value + confidence + source_id + observed_at
```

更新画像时不会修改或删除旧观察。这样可以重新计算任何时间点，也能区分“以前经常兑现”和“最近连续失约”。

## 双时间尺度

每条观察的权重：

```text
confidence × 0.5 ^ (age_days / half_life_days)
```

- 短期半衰期：21 天；
- 长期半衰期：240 天。

如果短期与长期差距至少 0.28、近期证据质量足够且至少有 3 条观察，标记 `changing=true`。变化时当前值更偏近期；平稳时更偏长期。

## 观察的维度

- `responsiveness`：回应意愿和大致延迟；
- `initiative`：是否主动推进；
- `follow_through`：承诺有没有兑现；
- `memory_care`：是否记得过去的重要细节；
- `warmth`：互动结果的总体热度。

这些值都在 `[-1, 1]` 内，并带单独的证据质量。界面只在置信度达到门槛后展示结论。

## 策略后验

本地规则从用户实际发送的文字识别短句、提问、玩笑、直接表达、推进见面和表达关心。每类策略以 `Beta(1.5, 1.5)` 为先验，成功、普通、失败和没回复分别映射到有界结果值。

旧反馈按 120 天半衰期衰减。最终 multiplier 还会乘置信度，并被限制在靠近 1 的范围内，因此它只改变推荐顺序，不会变成“永远必须这样说”的规则。

## 文字和行动的证据权重

`responseEvidenceWeight` 综合兑现和记得细节的观察；`actionEvidenceWeight` 保持更高的基础优先级，并在表态可靠度下降时进一步提高。

这两个数会进入系统上下文，但提示词明确要求：

- 当前明确事实优先；
- 近期变化和长期习惯分开；
- 冲突证据保留冲突；
- 不把画像当作对方内心的事实。

## SQLite

数据库启用 WAL、`synchronous=NORMAL`、外键和 busy timeout。截图记忆使用唯一键 `(profile_slug, memory_id)`，反馈和观察使用 UUID 追加。

`sqlite-vec` 可用时建立 384 维 `vec0` 表并按 profile partition 查询。扩展不可用时，向量仍以 Float32 BLOB 存在普通表里，由 TypeScript 做精确余弦计算。降级不会改变数据格式。

## 不做什么

- 不根据一次反馈更新永久性格；
- 不自动读取微信后台或监控用户；
- 不把反馈上传到项目自己的服务器；
- 不在本机微调基础模型；
- 不声称画像等于真实心理状态。
