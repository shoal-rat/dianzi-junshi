# 发布桌面安装包

v4 开始，普通用户不需要安装 Bun、Rust，也不需要使用终端。GitHub Release 会直接提供：

| 系统 | 推荐下载 | 备注 |
| --- | --- | --- |
| Windows 10/11 | `电子军师_*_x64-setup.exe` | NSIS 安装程序；也同时生成 MSI |
| macOS Apple Silicon | `电子军师_*_aarch64.dmg` | M1、M2、M3、M4 及更新芯片 |
| macOS Intel | `电子军师_*_x64.dmg` | 较旧的 Intel Mac |
| Linux x64 | `.AppImage` 或 `.deb` | AppImage 免安装；Debian/Ubuntu 可用 DEB |
| Linux ARM64 | `.AppImage` 或 `.deb` | ARM 笔记本和开发板 |

## 怎么发布

1. 同步以下三个版本号：
   - `app/package.json`
   - `desktop/package.json`
   - `desktop/src-tauri/tauri.conf.json` 和 `Cargo.toml`
2. 更新 `CHANGELOG.md` 和 `docs/releases/` 下的版本说明。
3. 推送同版本标签，例如 `v5.0.0`。
4. `.github/workflows/release-desktop.yml` 会在五种目标环境中构建安装包，先创建一个 Draft Release。
5. 下载并实测安装包，然后再手动发布 Draft。

也可以从 GitHub Actions 页面手动运行工作流，用于只验证构建。

## 签名与系统警告

工作流在没有证书时也能生成测试安装包，但公开发布前应配置签名：

- macOS：配置 `APPLE_CERTIFICATE`、`APPLE_CERTIFICATE_PASSWORD`、`APPLE_SIGNING_IDENTITY`、`APPLE_ID`、`APPLE_PASSWORD` 和 `APPLE_TEAM_ID`，完成 Developer ID 签名与 notarization。
- Windows：使用受信任的代码签名证书；否则 SmartScreen 可能显示“未知发布者”。
- Tauri updater：如启用自动更新，再配置 `TAURI_SIGNING_PRIVATE_KEY` 和密码。私钥只能放在 GitHub Secrets，不能提交进仓库。

签名不是装饰。它让用户确认安装包确实来自项目维护者，并避免系统把下载的 App 当作损坏文件。

## 桌面结构

桌面程序由两个小部件组成：

1. Tauri 原生窗口，负责安装包、窗口生命周期、单实例和系统 WebView。
2. Bun 编译的本地后端 sidecar，负责模型连接、SQLite、向量检索和截图任务。

sidecar 只监听随机的 `127.0.0.1` 端口。窗口退出时，Tauri 会同时结束后端进程。用户电脑不需要另装 Bun、Node.js、Rust 或数据库。

`sqlite-vec` 原生库会在通过系统加载校验的平台上随安装包使用。macOS 发布包目前优先使用兼容的精确余弦检索，避免第三方动态库破坏签名和 notarization；其他无法加载扩展的环境也会自动降级，资料和排序逻辑仍然可用。
