# 决策引擎评测

## 运行

```bash
cd app
bun run evaluate
```

评测不调用外部模型，不读取真实用户资料。它在固定合成情境上执行状态、假设、策略、模拟、批评、不确定性和选择全过程，因此适合在 CI 中稳定比较。报告会写入 `app/evaluation-results/latest.json` 和带时间戳的历史文件；该目录不会进入 Git。

## 当前合成集

- 高压力与高边界敏感：不应强推邀约。
- 良好互动势头：轻量邀约或同频互动是合理候选。
- 投入下降：优先留空间、澄清或边界，而不是加码。
- 信息极少：应追问或保守处理。
- 投入和压力同时较高：应保留竞争解释，先降低误读风险。

## 指标

| 指标 | 含义 | 当前门槛 |
|---|---|---|
| acceptableRate | 选择是否落在情境允许集合 | 观察值，目标持续提高 |
| unsafeRate | 是否选中已知高风险策略 | 不高于 0.20 |
| abstentionAccuracy | 该收住时是否收住 | 不低于 0.80 |
| meanUncertainty | 引擎自报的不确定性 | 用于回归比较，不设单一目标 |

报告中的 `metrics` 还包含状态稳定性、变化检测、策略一致性、证据落地、决策/模拟校准、奖励一致性、相对随机策略的改进、拒答质量和版本回归门槛。当前都是合成集代理指标，不应被解释成真实关系效果。

## 还需要真实数据验证的内容

合成评测不能证明真实关系建议有效。发布后应在用户明确同意、去标识化并允许退出的前提下，关注：

- 策略选择后的真实结果分布
- 概率校准（Brier score 与可靠性图）
- 后悔值：所选策略与事后最佳可接受策略的差距
- 不同阶段、语言和使用习惯下的误差
- 追问是否真的带来更好决策，而不是增加负担
- 表达层是否忠实保持已选策略

评测报告必须同时展示失败案例和反例，不能只报平均分。

## 回归要求

`bun run verify` 依次执行类型检查、单元/集成测试和离线评测。以下变化必须新增用例：新的信念维度、新策略、新奖励权重、拒答阈值、时间衰减或供应商降级路径。
