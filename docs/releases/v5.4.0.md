# v5.4.0 · Image-First & One Token Space — 2026-07-12

## 只发截图，也帮你选

以前图片只在最后写文案时给大模型看，本地引擎对着空文本"盲决策"。现在截图是一等输入：

1. 读图器（视觉结构化调用）从截图读出——对方最后一条消息原文、按顺序的对话摘录、九维状态信号（含语气与潜台词）。
2. 这些观察进入同一条决策流水线：信念滤波、竞争假设、世界模型推演、批评器；检索查询与时间事实也使用读出的文本。
3. 用户不打字、只发一张图，照样得到几条能直接发送的方案和推荐理由。

支持面：Anthropic（强制 tool + image blocks）、GLM-4V（vision content）、Codex / Claude Code（只读截图路径）；读图结果带缓存与确定性回退，读不出时引擎按先验保守处理。观察验证器逐行容错——单条格式不对的信号被丢弃，不会让整批有效读取作废。

## 一套 token 空间

不再保留旧向量空间的兼容层：

- `embedding.ts` 基于词典分词（`Intl.Segmenter`）生成 384 维特征哈希向量；BM25 词法、素材词法重排与向量语义全部共用这一套 token。
- 启动迁移（schema v6）把已存储的素材向量按 retrieval 原文自动重嵌入，一次完成，幂等。
- 中英混排一致处理：「我们Friday见」→ 我们 / friday / 见；嵌入大小写归一、语序无关。

## 质量

- 测试 27 → 30：混排分词、统一嵌入语义一致性（跨语序/大小写余弦 > .95、相关性排序）、读图验证器。
- 端到端验证：合成聊天截图 → 一次视觉调用读出 10 条状态信号（主动性 0.8、压力 −0.1 等）→ 假设更新至 receptive 71% → 输出三条可发送方案与推荐。
- 离线安全评测保持全绿。

## 兼容性

- 素材向量迁移自动进行；决策报告、API、前端结构不变（回复输入框提示更新）。
- `PipelineInput` 新增 `images` / `localImagePaths`；无图时行为与 v5.3 完全一致。
