# v5.3.0 · Temporal CNN — 2026-07-12

结构响应头只看得到当前状态的概要，对轨迹形状是盲的：升到 0.5 的投入和从 0.9 跌到 0.5 的投入，在概要里是同一个数，后续却往往完全不同。v5.3 给世界模型加上第三个部件——一个直接读原始观测时间线的时序卷积网络。

## 预测器本体（`decision/neural.ts`）

- 输入 10×16 观测网格：九个信念维度按 2.8 天分桶的加权均值（覆盖最近 45 天），加一条观测密度通道——沉默间隔在网格里是可见信号。
- 两层一维卷积（核宽 3，感受野约 14 天）→ 时间维全局平均池化 → 拼接动作特征 φ(a) 与体制后验 q(h) → 两层全连接 → 四类 softmax。共 1252 参数。
- 纯 TypeScript 实现前向与反向传播（反向经数值梯度校验），Adam 优化、类平衡权重、标签平滑、L2；零依赖，无 ONNX/PyTorch，安装包体积不变。
- 种子确定：同样的数据训练出逐字节相同的权重，事件回放可完整重建。

## 诚实门控

CNN 不凭架构新颖获得话语权。每次真实结果触发重训：按时间顺序留出最后 25% 的结果做对照，只有它的保留集对数损失优于当时响应头 0.02 nats，才获得混合权重 min(0.5, n/(n+12))；混合在推演树根节点逐分支线性实现，保持边际一致。数据不足（少于 8 条结果）或不比头更准时，权重为零，引擎行为与 v5.2 完全一致。

## 可见、可审计

- 决策网络图新增「时序卷积 CNN」节点：混合权重、参数量、训练样本、保留集优势，以及它对四个回应类别的推拉边。
- 诊断接口新增 `neuralPredictor`：样本数、参数量、保留集对照与当前权重。

## 基准

`bun run evaluate` 内置对照基准（种子固定可复现）：构造「当前概要相同、只有轨迹形状不同」的合成序列——升温收于 0.5、一直平稳在 0.5、降温收于 0.5 且末端沉默、压力震荡上行。概要结构头在这类数据上保留集对数损失 1.61 nats；时序 CNN 约 0.05 nats。概要看不出的形状，卷积看得出。

## 词典分词

检索前先分词：`Intl.Segmenter`（ICU 词典分词，JavaScript 引擎内建、零依赖、全本机）把句子切成真实词，BM25 的 IDF 统计从此落在词上；Han 字符 bigram 仅作召回补充，无分词器环境自动回退 n-gram。已存储的素材向量保持原 token 空间，升级不需迁移。

## 按人自适应的时间常数

21/240/120 天不再全局写死：引擎按每个档案相邻互动的中位间隔缩放时钟（clip 到 [7,60] / [90,480] / [45,360] 天）。天天聊的人保持标准档，几天一聊的人自动拉长——同样的信息量由同样多的往来承载。时钟随决策报告持久化，学习与回放使用记录当时的时钟；决策网络的不确定性节点会显示本档案的 τ。
