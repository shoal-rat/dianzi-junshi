# v5.6.0 · Memory Bank — 2026-07-12

长期记忆从「一张截图一条卡片、固定权重检索」升级为一个可检查、可管理的记忆库。

## 两阶段检索（优先级 1）

固定权重（0.56 向量 + 0.27 词法 + 0.14 重要度 + 0.03 时间）对所有问题一视同仁，效果偏弱。现在改为：

- **阶段一·广召回**：五路独立检索凑 30–50 个候选——关键词（BM25）、哈希向量、可选语义向量、人物/实体、日期，再加关联记忆扩展。
- **阶段二·重排**：先判断问题类型（事实 / 日期 / 人物 / 趋势 / 一般），按类型给语义、事实匹配、来源可靠度、时间适用性不同权重；停用、被推翻、近似重复的记忆在这里降权。
- **阶段三·多样化**：MMR + 相似度抑制，六个结果位覆盖六件不同的事，而不是被三张几乎一样的截图占满。

「她生日是哪天」偏重精确词、人物和日期；「她最近是不是疏远我了」偏重语义、时间趋势和多条证据——同一个库，不同问题走不同路径。

## 可选本机语义嵌入（优先级 2）

特征哈希看不出「态度变冷」和「没以前热情」是一个意思。现在：

- 装了本机嵌入模型（自动探测 Ollama，或填一个 OpenAI 兼容 `/embeddings` 端点，如 LM Studio / llama.cpp / vLLM）→ 用语义向量做混合检索。
- 没装 → 回退特征哈希，零依赖模式完全照常工作。
- 设置里三档切换：自动 / 自定义 / 关闭；向量按 (文本, 模型) 缓存在 SQLite。

特征哈希没有被删掉——它是离线底座，也擅长精确词匹配。语义只是把它从「最强语义检索」降级为「其中一路」。

## 事件记忆 + 截图去重（优先级 3）

一次约会常常分散在几张截图里（提议 → 确认时间 → 改地点 → 事后反馈）。现在相邻截图会聚合成一个事件记忆：

```json
{ "eventType": "meeting", "participants": ["user", "她"], "startedAt": "...",
  "status": "completed", "facts": [...], "sourceMemoryIds": ["a.png","b.png","c.png"] }
```

「上次那次见面是怎么约的」会命中一个完整事件，而不是几张散落的截图。字节级重复的截图在送进视觉模型前就被合并；近似重复标记后在检索里降权。

## 结构化事实（优先级 4）

事实不再是字符串数组，而是带生命周期的对象：

```json
{ "text": "她说她周五有空", "type": "availability", "confidence": 0.93,
  "observedAt": "2026-07-10", "sourceImage": "xxx.png",
  "sourceRegion": { "x": 0.1, "y": 0.4, "width": 0.7, "height": 0.2 }, "status": "active" }
```

七种类型区分「周五有空」（一次性，不该变成永久偏好）和「不喜欢吵的地方」（长期偏好），还有用户猜测、模型推断、明确约定。较新的事实会把冲突的旧事实标为 superseded / contradicted，过期信息不再左右后续回答。旧的字符串事实由迁移自动包装成结构化对象，保留来源图和观测时间。

## 记忆中心（优先级 5）

侧栏新增「记忆中心」：每条记忆显示摘要、来源截图、提取的事实、时间、重要度、关联记忆、被检索次数，以及被用到的原因。用户可以改、停用或删除记错的记忆和事实。

每条用到长期记忆的回复都带一个「📎 用到 N 条长期记忆」入口，右侧面板逐条列出为什么被选中（例如「问题类型『日期』；命中：关键词；事实匹配『她生日是5月18号』」）。AI 读错的摘要不该在用户看不见、改不了的情况下继续影响判断。

## 长期与短期分家

「长期记忆」只该装长期的东西。新增 `attribute` 持久属性类型（生日、名字、职业、老家——生日虽然是日期，但它不过期）；one_time / availability 短期安排在时机过去后自动转为 `expired`：写明日期的过了那天即失效，只写「这周六」这类相对时间的超过 30 天视为已过。过期事实退出检索评分、提示词与决策证据，但保留在记忆中心，可查可恢复。

## 决策网络可交互展开

决策网络小图右上角「⤢ 放大」（或直接点击图面）打开大图模态：滚轮围绕光标缩放、拖动平移、点任意节点查看激活值、参数和最强流入/流出贡献，一键复位视图。

## 迁移与兼容

- SQLite schema v7：material_memories 增补 content_hash / status / duplicate_of / retrieval_count / semantic_vector 等列；新增 event_memories、embedding_cache、memory_retrieval_log 表。列增补逐个 guard，重跑与全新库都安全。
- 旧字符串事实一次性包装为结构化对象（幂等）；旧素材向量沿用 v6 的重嵌入。
- 全部 37 个测试通过；离线安全评测保持全绿（acceptable 1.0 / unsafe 0 / abstention 1.0）。
