# 电子军师 v4.0.0 · Adaptive Desktop

发布日期：2026-07-11

## 一句话版本

电子军师现在是可以直接下载安装的桌面 App；它不只记住过去，还会从真实后续中分辨“长期习惯”和“最近变了”。

## 下载安装

Release 会提供：

- Windows：Setup.exe 和 MSI；
- macOS：Apple Silicon 与 Intel DMG；
- Linux：x64 / ARM64 的 AppImage 和 DEB。

安装包内含 Tauri 窗口、Bun 后端和 SQLite。用户不需要另装开发工具或数据库。

## 新的结果反馈

每个回复方案旁边都有「记录后来结果」。用户可以记录：

- ta 是接住了、普通回复、变冷，还是没回；
- 大约多久回复；
- 有没有继续聊或主动推进；
- 答应的事情是否兑现；
- 是否记得过去的重要细节；
- ta 实际回复的原文。

这些是观察事件，不是永久标签。App 同时维护近期和长期估计；旧证据会衰减，重复结果才会提高置信度。

## 为什么需要时间画像

近期研究显示，单纯把全部历史塞进提示词并不能可靠识别人物变化。电子军师因此显式区分：

1. 短期状态：约 21 天半衰期；
2. 长期习惯：约 240 天半衰期；
3. 近期是否与长期产生有置信度的偏离；
4. 不同表达策略在这个档案上的真实结果。

如果样本少，界面会说“先不急着下结论”。如果近期确实变了，会显示变化而不是偷偷重写旧画像。

## 数据库和检索

新增 SQLite WAL 数据库：

- `material_memories`：截图记忆卡和向量；
- `feedback_events`：真实后续；
- `trait_observations`：带时间与置信度的观察；
- `schema_migrations`：可升级的数据结构。

`sqlite-vec` 能安全加载时负责向量 KNN；否则使用精确余弦检索。两条路径都继续结合关键词、重要度、时间和记忆关联。

## 桌面架构

Tauri 从系统分配一个空闲端口，启动 Bun sidecar，再打开本机 WebView。sidecar 只绑定 `127.0.0.1`，窗口退出时一并结束。单实例插件避免重复打开多个后台。

Linux 安装包把独立 Bun 后端压缩为应用资源，第一次打开时原子解压到当前用户缓存，之后直接复用。这样既避开 AppImage 对静态 sidecar 的错误动态依赖扫描，也不会要求用户安装 Bun。

GitHub Actions 会在五种目标环境分别构建安装包并汇总进 Draft Release。公开发布前仍需要维护者配置 Apple notarization 和 Windows 代码签名。

## 研究和工程依据

- [PersonaMem](https://arxiv.org/abs/2504.14225)：动态用户画像仍是现有模型的明显弱点。
- [Memoria](https://arxiv.org/abs/2512.12686)：会话摘要与加权知识图结合的长期个性化框架。
- [sqlite-vec](https://github.com/asg017/sqlite-vec)：可嵌入 SQLite 的跨平台向量搜索。
- [Tauri sidecar](https://v2.tauri.app/develop/sidecar/)：把外部后端打进原生安装包。
- [Tauri GitHub release pipeline](https://v2.tauri.app/distribute/pipelines/github/)：多平台安装包与 Release 自动化。

## 升级边界

- v3 的档案和原始资料继续可用；JSONL 素材记忆会第一次读取时导入 SQLite。
- “学习”是有界重排，不训练或修改基础模型。
- 当前反馈需要用户主动记录，App 不会监控微信或其他聊天软件。
- macOS 发布构建在动态库签名不满足系统校验时使用精确向量降级，不牺牲启动可靠性。
