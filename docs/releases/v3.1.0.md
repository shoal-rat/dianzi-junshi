# 电子军师 v3.1.0 · Deep Memory

发布日期：2026-07-10

## 一句话版本

现在可以一次丢进一整批过往截图：文件逐个落盘，AI 在后台一张张读，进度看得见；很久以前的重要截图也能按语义重新找回来。

## 为什么做这版

关系上下文不是最近十几条消息就能说清的。一次见面有没有兑现、某个偏好是什么时候说的、一次冲突后来怎么收尾，这些证据可能藏在几个月甚至几年前的截图里。

简单地把所有图片塞进一次模型请求有三个问题：浏览器内存会迅速膨胀、模型上下文装不下、任何一张失败都可能拖垮整批。因此 v3.1.0 把「上传」「理解」「检索」拆成三个可恢复阶段。

## 用户会看到什么

1. 在新建档案里一次选择任意数量的截图。
2. 文件逐个保存到本机，进度条显示当前文件和数量。
3. 后台一次只让 AI 读取一张图。
4. 每张图完成后立即写入长期记忆索引，不必等整批结束。
5. 可以收起进度、关闭新建窗口或切换页面；任务继续运行。
6. 失败项可以单独重试，不重复上传成功项。

如果当前还是演示模式，截图会先安全保存在本机。选择支持看图的连接后，队列自动继续。

## 长期记忆怎么工作

每张截图会产生一张可解释的记忆卡：

- 简短摘要；
- 可回到原图核对的事实；
- 人物、时间、情绪和关键词；
- 0–1 重要度；
- 为未来检索补充的同义描述；
- 384 维本地向量；
- 与其他相似记忆的链接。

用户提出新问题时，排序会综合向量相似度、关键词重合、重要度和很弱的时间衰减。时间只占很小权重，因此旧的关系转折可以超过昨天的普通闲聊。

## 设计来源

- [A-MEM](https://arxiv.org/abs/2502.12110) 使用带上下文、关键词和标签的动态记忆笔记，并在记忆之间建立关联。
- [Microsoft GraphRAG Local Search](https://microsoft.github.io/graphrag/query/local_search/) 将语义相关实体、结构化关系和原始文本一起排序后放入上下文。
- [RAPTOR](https://proceedings.iclr.cc/paper_files/paper/2024/hash/8a2acd174940dbca361a6398a4f9df91-Abstract-Conference.html) 展示了多层摘要和检索对长材料整体理解的价值。

电子军师没有直接引入大型向量数据库或额外模型依赖。当前实现用 LLM 生成语义丰富的记忆卡，再在本机做固定维度向量和混合重排，保证 App 仍可单文件构建。

## 边界

- “数量不限”指产品不设置固定张数上限；实际吞吐仍取决于磁盘空间和 AI 额度。
- 摘要和向量可能遗漏信息，所以原图永远保留，并作为最终核对来源。
- 视觉整理会使用当前选择的 AI 服务；上传到本机不等于发送给模型，只有进入逐张分析时才会传给所选服务。
- 当前队列按档案串行处理，优先稳定和可恢复，而不是追求最大并发。

## 升级

拉取最新代码后重新启动即可：

```bash
git pull
cd app
bun start
```

旧档案无需迁移。新的 `material-jobs/` 和 `material-memories.jsonl` 会在第一次批量导入时自动创建。
