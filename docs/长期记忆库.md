# 长期记忆库

从 v5.6 起，导入的截图不再只是「一张图一条卡片」，而是一个可检索、可管理、可解释的记忆库。这份文档说明它的结构、检索方式、如何开启本机语义模型，以及如何管理记住的内容。

## 三种记忆

| 类型 | 来源 | 用途 |
|---|---|---|
| 截图记忆（material memory） | 每张导入截图经视觉模型整理 | 摘要 + 结构化事实 + 检索文本 + 向量 |
| 事件记忆（event memory） | 相邻截图自动聚合 | 把一次约会/一场矛盾的多张截图合成一个整体 |
| 结构化事实（fact） | 截图记忆里的字段 | 带类型、置信度、来源、生命周期的可校验事实 |

## 结构化事实

```json
{
  "id": "…",
  "text": "她说她周五有空",
  "type": "availability",
  "confidence": 0.93,
  "observedAt": "2026-07-10",
  "sourceImage": "xxx.png",
  "sourceRegion": { "x": 0.1, "y": 0.4, "width": 0.7, "height": 0.2 },
  "status": "active",
  "supersededBy": "另一条事实的 id（如被更新）"
}
```

类型（`type`）：

- `one_time` / `availability`：一次性安排，如「周五有空」。默认不会被当成长期偏好。
- `preference`：长期偏好，如「不喜欢吵的地方」。
- `agreement`：双方明确约定。
- `speculation`：用户的猜测。
- `inference`：模型的推断，不是对方明说的。
- `observation`：中性观察（兜底）。

生命周期（`status`）：`active` → 生效；`superseded` → 被更新的事实取代；`contradicted` → 被相反的事实推翻；`retired` → 用户手动停用。较新的事实与旧事实在文本高度重合且出现「否定翻转」或「日期变化」时，旧事实自动转为 superseded/contradicted，不再影响后续回答。

## 两阶段检索

1. **广召回（30–50 候选）**：关键词 BM25、哈希向量余弦、可选语义向量、人物/实体命中、日期命中，再加关联记忆扩展。
2. **重排**：先判断问题类型（事实 / 日期 / 人物 / 趋势 / 一般），按类型加权语义相关、事实匹配、来源可靠度、时间适用性；停用 / 被推翻 / 近似重复的记忆降权。
3. **多样化（MMR）**：抑制相似记忆，让有限的结果位覆盖不同事件与证据。

每条被选中的记忆都会记录「为什么」，可在右侧面板和记忆中心查看。

## 可选：本机语义嵌入

零依赖的特征哈希是底座，但它看不出「态度变冷」≈「没以前热情」。装一个本机嵌入模型即可增强语义检索：

### 方式一：Ollama（推荐，自动检测）

```bash
# 安装 Ollama 后拉一个多语言嵌入模型
ollama pull bge-m3         # 或 nomic-embed-text / mxbai-embed-large / all-minilm
```

设置 → 语义嵌入 → 选「自动检测本机 Ollama」。应用会探测 `http://127.0.0.1:11434` 并挑一个可用的嵌入模型。

### 方式二：自定义 OpenAI 兼容端点

LM Studio、llama.cpp server、vLLM 等提供 `/v1/embeddings` 的服务：

设置 → 语义嵌入 → 选「自定义」，填 Base URL（如 `http://127.0.0.1:1234/v1`）和模型名。

### 方式三：关闭

始终使用特征哈希。应用行为与旧版一致。

无论哪种方式，模型都跑在你自己的机器上，向量按 (文本, 模型) 缓存在本机 SQLite；探测失败或调用出错都会自动回退哈希。

环境变量 `DJ_DISABLE_SEMANTIC=1` 可强制关闭（测试用）。

## 记忆中心

侧栏「记忆中心」可以：

- 查看每条记忆的摘要、来源截图、结构化事实、时间、重要度、被检索次数和最近被用到的原因。
- 修改单条事实的状态（生效 / 已被更新 / 已被推翻 / 停用）。
- 停用整条记忆（不再进入检索）或彻底删除（原始截图文件仍保留在磁盘）。
- 查看和删除聚合出的事件记忆。

## 数据与迁移

- schema v7：`material_memories` 增补 `content_hash`、`status`、`duplicate_of`、`retrieval_count`、`last_retrieval_reason`、`semantic_vector`、`semantic_model`；新增 `event_memories`、`embedding_cache`、`memory_retrieval_log` 表。
- 旧字符串事实一次性包装成结构化对象，保留来源图和观测时间；旧向量沿用 v6 的重嵌入。
- 迁移幂等，旧库直接升级，无需手动操作。
