# 对象档案：建档、模板、朋友圈分析、批量导入

本文件管对象档案（partners/{slug}/）的全部生成和维护。目录：

1. 建档流程 — 代号唯一必填，要材料不要问卷
2. 档案模板 — profile.md 的完整结构
3. 怎么填 — 各区块提取要点 + 填充规则
4. 朋友圈/社交动态怎么看 — 单图留痕与画像
5. 批量导入 — tools/import_folder.py 分类与处理
6. 图片读取规则 — 全尺寸原图，不看缩略图

## 一、建档流程

目标：用最少的问题让用户马上拿到第一条能发的建议。用户只会打字、不想填表，能自己看出来的就别问。

开场（一句话就够）：

```
我是电子军师。把 ta 的聊天截图丢给我，我读完告诉你 ta 什么意思、有没有戏，再给你几条能直接发的回复。
先给 ta 起个代号，剩下的边聊边补。
```

### 唯一必填：代号

非空即可。slug 规则：中文转拼音、英文小写、空格转下划线。代号和已有对象撞了，让用户加后缀（小美2、公司的小美），别覆盖旧档案。

### 然后：要材料，不要问卷

让用户把聊天截图、聊天记录或朋友圈截图丢进来；一句话说说 ta 是谁、你俩什么情况也行。

- **有材料**：自己从里面读，别逐项审问——大概阶段（互动热度、谁主动、怎么称呼）、ta 的性格和说话习惯、对方兴趣高低、用户自己的说话风格（之后照着用户发过的话写回复）。看不出来的标 `[待观察]`，聊着聊着自然补上。
- **没材料**：只补问下面两个，其余全部留空，别多问。

补问一：到哪一步了

```
┌ 你俩现在大概到哪一步？（回数字，不确定就回“跳过”）
│ 1 刚认识   2 暧昧   3 在追   4 刚在一起   5 在一起挺久了   6 闹别扭
└ 回数字，跳过也行
```

映射到 `references/strategy.md` 的阶段和油腻度上限。跳过就先按“暧昧”估，之后用聊天证据修。

补问二：要不要反舔狗模式

```
┌ 要不要开反舔狗模式？（回数字）
│ 1 开：没戏时直接拉住你，不陪你自我感动
│ 2 不开（默认）：也提醒兴趣低，但语气缓一点
└ 默认不开，直接回数字
```

### 建完就上手

由桌面 App 在本机数据目录建立独立档案并写入 SQLite 索引，立刻给第一条建议或回复，别让用户干等。以后可以从左侧切换或新建档案。

## 三、怎么填

按模板区块的提取要点：

- **先破题名**：代号、文件夹名、截图名只是标签。建档必须独立判断：是否确认关系、是否双向主动、有没有反问接球补细节、有没有邀约替代时间和行动兑现、有没有承诺和公开选择、有没有止步证据。档案名叫"热恋"但证据只有单向主动无见面兑现，就写清楚"标签是热恋，证据不支持热恋"。
- **沟通风格**：看密度、长度、语气词、标点、表情包类型（是否易被代际误读）、语音、称呼；从原材料抄真实例句，别转写。
- **情感需求**：爱的语言五类（肯定言辞/精心时刻/礼物/服务行动/身体接触）挑主次；被回应看 ta 对速度和长度的实际反应；被看见看 ta 分享和倾诉时要什么（夸/建议/共鸣/陪着）。
- **行为模式**：撒娇、生气（冷处理还是连发"？？？"）、不开心信号、开心的样子、拐弯抹角的试探习惯，全写具体表现，不写抽象标签。
- **性格底色**：涉及深判断、约会设计、气质选择时才补全；每条轴带证据，轴定义见 `references/strategy.md`。
- **兴趣度**：必须写四维分，聊天甜 ≠ 承诺和行动；ta 是真有意思、礼貌回复，还是只把用户当情绪/资源支持，要分开判。证据不足标 [待观察]，不为给答案硬判。规则见 `references/reading.md`。
- **追法与拉扯**：类型、性别脚本（证据不足不按性别套）、拉扯强度 0-3、加码/暂停窗口，见 `references/strategy.md`。
- **微信息**：MBTI 只记"自称/提到"，不当人格真理；前后不一致只记次数和证据；未知梗按 `references/memes.md` 先查证再写。
- **潜台词**：记录 ta 个人化的表面话/真实需要两层（如"你好烦啊"可能是撒娇），通用解读只是起点。

填充规则：

1. 全部基于聊天记录或用户描述，不得虚构
2. 潜台词字典必须结合具体原材料，不用通用解读
3. 信息不足标 `[待补充]`，观察类标 `[待观察]`
4. 字段要具体："很敏感"要写成"发消息15分钟没回就会发'?'"
5. 朋友圈只当展示面，不当完整人格
6. 兴趣度基于连续证据和四维分；反舔狗开启时也要给证据再建议止损
7. 代号只当标签，不当阶段事实
8. 每次大分析或档案重估更新 `versions/analysis_vN.md`，并在档案登记版本
9. 图片结论必须能回指 `materials/image_observations.jsonl` 的图片编号

## 四、朋友圈/社交动态怎么看

朋友圈是对外展示面，不是完整人格。只输出"可能的展示倾向"，不下诊断式结论。

输入：朋友圈/小红书/微博/Instagram 截图、动态文案、照片、既往聊天或 `tools/wechat_parser.py` 报告。

**必须直接看图片，不只做 OCR。** 朋友圈大部分信息在图里不在字里：妆容、穿搭、滤镜、场景、同框的人、评论区。按本文件「图片读取规则」看原图全尺寸。一次收到多张图不许抽样只看"关键图"，每张都要留痕。

### 单图留痕

每张图先写一条记录进 `partners/{slug}/materials/image_observations.jsonl`，再综合判断。字段：

- `image_id`（沿用导入模板编号，没有就临时编号）· `visible_context`（朋友圈/聊天截图/自拍/合照/评论区/未知）
- `initiator`（聊天截图记谁主动开启本轮；非聊天图写 not_applicable）· `receiver_response`（接球/部分接/错过/转移/礼貌回应）
- `has_counter_question`（有没有反问或把话题带回用户）· `has_invite`（有没有邀约、时间地点是否具体）
- `action_fulfillment`（已兑现/待兑现/失败/未知/不适用）
- `warming_signals`（主动补细节/接梗/分享生活/给时间等）· `cooling_signals`（只答不问/拒约不给替代时间/长时间不回/转移话题等）
- `visible_evidence`（只写图上看得到的，不写脑补）· `confidence`（高/中/低）

综合结论必须引用记录，例如"img_0004 有接梗但无反问，img_0011 拒约不给替代时间，所以聊天甜度高于行动兑现"。

### 展示主题

| 类型 | 常见内容 | 聊天切入 |
|------|----------|----------|
| 生活记录 | 吃饭、通勤、日常小事 | 接日常细节，轻松共鸣 |
| 审美品味 | 咖啡、展览、音乐、穿搭、旅行 | 夸具体品味，不空泛夸人 |
| 成就展示 | 工作、学习、证书、健身 | 真诚认可，轻松不跪舔 |
| 情绪表达 | 深夜文案、emo、歌词 | 先接情绪，不急着撩 |
| 社交活跃 | 聚会、朋友、活动 | 展示自己的生活，不查岗 |
| 低展示/神秘 | 很少发、内容克制 | 低油腻少问隐私，多看聊天反馈 |

类型假设只写"可能偏向"并给证据；不因性别、穿搭或单张照片判断"好追/不好追"。

### 外貌/妆容/穿搭

提取：展示方式（自拍/他拍/背影/局部/不露脸/合照）、妆容（淡/浓/清冷/甜妹/欧美/韩系/素颜感/口红色系）、穿搭（运动/通勤/甜酷/极简/学院/辣妹/户外/复古）、拍照滤镜和氛围感。用途：夸具体审美不空泛夸"好看"、选约会地点/礼物/花、帮用户展示同频的自己。

### 互动方式与玩家信号

评论区看：谁常评论（朋友/同事/异性/固定圈子）、ta 回复语气（热情/冷淡/玩梗/礼貌/撒娇）、是否公开暧昧、是否喜欢被起哄。别把评论区一个人的玩笑当事实。

玩家信号（平台人设型、评论区暧昧型、朋友圈定向投喂、节日借势、多线排班、模板化高情绪价值）：识别表和对应测试动作见 `references/reading.md`。

### 微信息提取

MBTI（只记自称/提到）、星座玄学（信不信/当梗玩）、口头禅+情境、说话置信度（确定/犹豫/开玩笑/反复横跳）、前后不一致次数（不直接定性撒谎）、礼物偏好、忌口和生活习惯、表情包/梗偏好（未知梗按 `references/memes.md` 先查证）。写入 profile.md 时标来源和置信度。

### 可聊话题 3 类

轻话题（开场不冒犯）、深一点的（聊过几轮后推进）、不建议碰（隐私/身材/收入/前任/定位/过度评价外貌）。

### 防文艺脑补

文案深情、照片氛围暧昧、评论区玩梗，都不等于对用户有兴趣。判断"ta 对用户有没有意思"只回到行动证据：有没有私聊找用户、有没有反问补细节、有没有推进见面或给替代时间、有没有把公开互动转成私下明确安排、有没有公开选择用户而不是对多人放同款信号。

### 输出格式

```markdown
## 朋友圈画像
**可见证据**：{截图/文案中的具体元素}
```

## 五、批量导入

用户只给一个文件夹路径，不用自己分类。触发：`/import-folder 路径`、`/import 路径`、"资料都在这个文件夹"。

流程：

1. 运行 `tools/import_folder.py --path "{folder}" --output partners/{slug}/materials/import_manifest.json`。
2. 读生成的 Markdown 摘要和 `image_observations.jsonl` 空模板。
3. 按类型处理（下表）。
4. 归类后最多问用户 1 个确认问题：优先分析"聊天记录""朋友圈画像"还是"邀约策略"。

分类标准（工具输出的全部类别）：

| 类别 | 特征 | 后续处理 |
|------|------|----------|
| `chat_text` | 文本且名称含 chat/wechat/微信/聊天/记录/qq/message | `tools/wechat_parser.py` 或直接分析：兴趣度、口头禅、邀约线索 |
| `moments_image` | 图片且名称含 朋友圈/moments/动态/小红书/微博 | 多模态看展示主题和互动 |
| `appearance_image` | 图片且名称含 自拍/照片/穿搭/妆/头像/selfie | 多模态看外貌展示、妆容、审美 |
| `screenshot` | 图片且名称含 screenshot/截图 或聊天类关键词 | 先多模态判断内容再归类；聊天图必记谁主动、接球、反问、邀约、降温信号 |
| `notes` | 文本/文档且名称含 note/备注/画像/印象/偏好 | 提取用户描述和关键事实 |
| `text_other` | 其它文本文件 | 读取内容后再归类 |
| `document` | .docx/.pdf 等文档 | 平台支持就读；不支持请用户导出文本 |
| `image_unknown` | 没命中关键词的图片 | 同样逐张多模态看，看完归类 |
| `unknown` | 无法判断 | 保留路径，等用户确认 |

多模态要求：**每张图片都要有一条 jsonl 记录，不许抽样**。工具先生成空模板，看完图必须更新字段，不能只在结论里口头概括。图片类材料不能只靠文字——看内容（人像/场景/构图/同行人）、看文案、看互动、看连续性（避免单张过度推断）、看行动证据（聊天截图优先看主动/反问/接球/邀约/兑现，别被文案甜度带偏）。所有判断写"可见证据"。

导入摘要输出：文件夹路径、各类别数量、建议优先级（1-2 条带理由）、下一步（"我会先处理 {类别}，想改优先级直接说"）。

## 六、图片读取规则

- 一律读原图全分辨率，别拿缩略图、预览图或压缩图下判断。时间戳、昵称、气泡小字、评论文字、妆容配饰都藏在缩略图会糊掉的那层；糊一层就容易把"谁主动、有没有反问、有没有兑现、是不是同一个人"读错。
- 导入清单和 `image_observations.jsonl` 里存的是原图 `path`，认原图，别分析任何缩略版本。
- 长截图、高清照片、多图拼接：分区域、分段放大看，别靠一次缩小后的整图下结论。
- 关键细节拿不准就裁剪/放大那一块再确认；真看不清就说看不清，请用户发更大更清楚的原图，别靠糊图硬猜。
