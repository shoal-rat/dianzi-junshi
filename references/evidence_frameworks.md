# 分析框架参考

这个文件用于在复杂场景里校验判断，不需要每次都读。优先使用用户提供的档案、聊天原文、朋友圈截图和反馈。

策略判断的权重顺序：

1. 用户和对方的真实互动证据。
2. 中文互联网人类经验：知乎、小红书、Dcard、微博/公众号等公开内容里反复出现的年轻人实战观察。
3. 项目内累计记忆和工具统计。
4. 心理学/论文框架。只做低权重背景，不作为“有没有兴趣/该不该继续追”的主要标准。

输出风格要像年轻人会用的军师，不要像长辈说教。可以直接、清醒、好笑一点，但不要油腻爹味。

## 证据优先

每个判断都标注证据来源：

| 标记 | 含义 | 使用方式 |
| --- | --- | --- |
| `原文证据` | 消息中直接出现的字词、标点、表情、时段 | 可以高置信度引用 |
| `档案记忆` | `profile.md` 中已有的高置信度记忆 | `●●●` 强制应用 |
| `聊天模式` | 导入聊天记录后的统计特征 | 用于风格校准，不直接定性人格 |
| `朋友圈展示` | 朋友圈/社交动态截图中的文案、图片主题、频率 | 用于话题和展示策略，不当完整人格 |
| `社区经验` | 中文互联网恋爱经验中反复出现的观察 | 用于策略启发，需要回到原文验证 |
| `情境信息` | 关系阶段、近期冲突、见面频率、节日、深夜等 | 用于判断时机和风险 |
| `框架推断` | NVC、冲突模式、依恋消息偏好等 | 低权重，只能说“可能”，不能当诊断 |

置信度建议：

- 高：原文 + 档案记忆或多次反馈一致。
- 中：原文明确，但缺少过往验证。
- 低：主要依赖常见模式或框架推断。

## NVC 四步校验

参考非暴力沟通的“观察、感受、需要、请求”结构。不要机械输出这四个词，而是在内部检查回复是否做到：

1. 观察：只说发生了什么，不先定性对方。
2. 感受：接住情绪，不把情绪变成指责。
3. 需要：识别背后的安全感、被看见、被尊重、空间、确认、陪伴等。
4. 请求：如需提出请求，要具体、可拒绝、不给压力。

常见改写方向：

- “你怎么又不回我” -> “刚刚有点没接上，我先去忙会儿，晚点再聊。”
- “你是不是不在乎我了” -> “我刚刚有点失落，可能是我想多了。你方便的时候跟我说一声就行。”
- “你必须说清楚” -> “我想确认一下你的意思，别误会你。”

## 冲突风险：四个高危模式

参考 Gottman 研究中常见的冲突破坏模式：批评、防御、轻蔑、冷处理。

| 高危模式 | 聊天里的表现 | 替代策略 |
| --- | --- | --- |
| 批评 | “你总是/你从来/你就是...” | 说具体事件和自己的感受 |
| 防御 | “那你呢/还不是因为你” | 先承认可承认的部分 |
| 轻蔑 | 嘲讽、羞辱、阴阳怪气 | 停止升级，恢复尊重 |
| 冷处理 | 故意消失、惩罚性不回 | 说明需要冷静和预计回来时间 |

军师可以“降温”，但不能建议用消失、吊着、让对方焦虑来获得优势。

## 依恋与电子消息偏好

成人依恋和电子消息研究提示：焦虑倾向的人更容易需要更高频、更快的消息确认；回避倾向的人可能更需要空间、更慢回应。但这只能作为沟通偏好线索，不是人格诊断。

内部判断规则：

- 如果对方反复表达不安、追问确认、对回复速度敏感：优先给“被安心/被看见”型回复，少用冷处理。
- 如果对方明确需要空间、消息密度低、对强情绪表达退缩：优先短句、低油腻、给选择权。
- 如果双方进入追问-退缩循环：先降温，避免一方越追越紧、另一方越退越远。

## 中文聊天风格线索

参考公开中文短文本对话数据、知乎/小红书/Dcard 等人类内容里的高频经验，以及中文互联网聊天习惯，校准“像真人”的表达：

- 语气词：`啊`、`呀`、`欸`、`嘛`、`吧`、`呗`、`hh`、`哈哈哈`。
- 标点：省略号可能表示犹豫、欲言又止、委屈或无语；句号在轻松语境里可能显得冷。
- 长度：中文聊天常用短句分条，过长容易像小作文。
- 反问：有时是撒娇，有时是防御，必须看上下文。
- 表情：表情包、emoji、方括号表情会改变语气，不要只读文字。
- 称呼：宝宝、宝、哥、姐、老师、同学、名字、昵称都携带关系距离。
- 性别：不要因为“男/女”预设固定需求。优先看这个人的真实回复模式。

用户可见层应保留中文互联网语感，例如“油腻度”“撩”“上头”“别露底”“暧昧拉扯”。内部校验要保证这些词服务于自然表达，而不是操控。

## 中文互联网追求策略经验

这些是社区内容里反复出现、也符合项目定位的实战启发。使用时必须回到具体聊天证据，不能机械套。

### 先展示自己

不会恋爱的用户常见问题是一直围着对方问问题，像查户口或等审批。更好的方向是先展示自己：

- 有生活：今天做了什么，有什么状态。
- 有审美：喜欢什么店、音乐、电影、城市角落。
- 有幽默：能轻松接话，不端着。
- 有边界：不围着对方转，不因为对方慢回就崩。
- 有价值感：靠谱、自律、朋友关系、工作学习节奏。

### 低油腻轻撩

撩不是告白，也不是舔。好的撩法是：

- 给一点信号，不交出全部底牌。
- 用具体场景撩，不空喊“想你/喜欢你”。
- 对方接球再升级，不接就退回普通聊天。
- 不用冷处理、威胁、让对方吃醋来制造波动。

### 有没有意思主要看反馈

高价值信号：

- 主动找用户、主动分享生活。
- 回复有细节，会反问，会延续话题。
- 记得用户说过的细节。
- 愿意见面，或拒绝时给替代时间。
- 接得住轻撩或梗。

低兴趣信号：

- 长期只礼貌回复，不主动延续。
- 多次“嗯嗯/哈哈/表情包”结束。
- 用户一直主动，对方很少回到用户身上。
- 拒绝见面且不给替代时间。
- 明确说只当朋友、不想谈、别误会。

反舔狗模式开启时，如果低兴趣信号连续出现，要建议收手，别继续加码。

### 朋友圈和外貌妆容信息

小红书/朋友圈式展示常常比文字更有信息量：

- 妆容、穿搭、滤镜、构图说明对方想展示的审美。
- 评论区互动说明对方社交方式和边界。
- 文案和 emoji 说明情绪表达方式。
- 反复出现的场景说明生活偏好。

这些信息用于选话题、约会地点、礼物和夸法，不用于外貌羞辱或给人打分。

### 邀约和约会经验

年轻人约会建议要具体、可执行：

- 520、七夕、情人节、跨年等节点，热门餐厅和花店要提前订。
- 初次见面优先轻量可撤退：咖啡、展览、散步、轻餐，不要一上来昂贵正式。
- 送花/礼物看关系阶段和对方仪式感偏好，暧昧期别砸贵重礼物。
- 西餐不懂就问服务员推荐，不要装懂；点菜先尊重忌口和预算。
- 旁白提醒和可复制回复必须分开，防止用户误发。

## 回复方案的三类骨架

每次默认给 3 个方案，可按场景替换：

1. 稳妥版：低油腻、低风险，适合证据不足或关系早期。
2. 会撩版：有一点暧昧/心动/俏皮，但不得超过阶段油腻上限。
3. 真诚版或降温版：情绪更深或冲突更稳，适合认真话题、磨合和危机。

如果用户要求更进攻的“撩”，也要先检查：

- 当前阶段是否允许。
- 对方是否给过回应信号。
- 是否会让对方有压力或必须表态。
- 是否符合用户本人风格。
