# 分析框架参考

这个文件用于在复杂场景里校验判断，不需要每次都读。优先使用用户提供的档案、聊天原文、朋友圈截图和反馈。

策略判断的权重顺序：

1. 用户和对方的真实互动证据。
2. 中文互联网人类经验：知乎、小红书、Dcard、微博/公众号等公开内容里反复出现的年轻人实战观察。
3. 项目内累计记忆和工具统计。
4. 常识背景，只做低权重校准，不作为“有没有兴趣/该不该继续追”的主要标准。

输出风格要像年轻人会用的军师，不要像长辈说教。可以直接、清醒、好笑一点，但不要油腻爹味。

## 证据优先

每个判断都标注证据来源：

| 标记 | 含义 | 使用方式 |
| --- | --- | --- |
| `原文证据` | 消息中直接出现的字词、标点、表情包/微信内置表情、时段 | 可以高置信度引用 |
| `档案记忆` | `profile.md` 中已有的高置信度记忆 | `●●●` 强制应用 |
| `聊天模式` | 导入聊天记录后的统计特征 | 用于风格校准，不直接定性人格 |
| `朋友圈展示` | 朋友圈/社交动态截图中的文案、图片主题、频率 | 用于话题和展示策略，不当完整人格 |
| `社区经验` | 中文互联网恋爱经验中反复出现的观察 | 用于策略启发，需要回到原文验证 |
| `情境信息` | 关系阶段、近期冲突、见面频率、节日、深夜等 | 用于判断时机 |

置信度建议：

- 高：原文 + 档案记忆或多次反馈一致。
- 中：原文明确，但缺少过往验证。
- 低：主要依赖常见模式。

## 中文聊天风格线索

参考公开中文短文本对话数据、知乎/小红书/Dcard 等人类内容里的高频经验，以及中文互联网聊天习惯，校准“像真人”的表达：

- 语气词：`啊`、`呀`、`欸`、`嘛`、`吧`、`呗`、`hh`、`哈哈哈`。
- 标点：省略号可能表示犹豫、欲言又止、委屈或无语；句号在轻松语境里可能显得冷。
- 长度：中文聊天常用短句分条，过长容易像解释太多。
- 反问：有时是撒娇，有时是防御，必须看上下文。
- 表情：表情包、微信内置表情、方括号表情会改变语气，不要只读文字。年轻人语境里部分内置表情可能有冷淡、敷衍或阴阳意味，必须看个人习惯。
- 称呼：宝宝、宝、哥、姐、老师、同学、名字、昵称都携带关系距离。
- 性别：不要因为“男/女”预设固定需求。优先看这个人的真实回复模式。

用户可见层应保留中文互联网语感，例如“油腻度”“撩”“上头”“别露底”“暧昧拉扯”。内部校验要保证这些词服务于自然表达。

## 中文互联网追求策略经验

这些是社区内容里反复出现、也符合项目定位的实战启发。使用时回到具体聊天证据，不机械套。

### 先展示自己

不会恋爱的用户常见问题是一直围着对方问问题，像查户口或等审批。更好的方向是先展示自己：

- 有生活：今天做了什么，有什么状态。
- 有审美：喜欢什么店、音乐、电影、城市角落。
- 有幽默：能轻松接话，不端着。
- 有节奏：不围着对方转，不因为对方慢回就崩。
- 有价值感：靠谱、自律、朋友关系、工作学习节奏。

### 低油腻轻撩

撩不是告白，也不是舔。好的撩法是：

- 给一点信号，不交出全部底牌。
- 用具体场景撩，不空喊“想你/喜欢你”。
- 对方接球再升级，不接就退回普通聊天。

### 有没有意思主要看反馈

高价值信号：

- 主动找用户、主动分享生活。
- 回复有细节，会反问，会延续话题。
- 记得用户说过的细节。
- 愿意见面，或拒绝时给替代时间。
- 接得住轻撩或梗。

低兴趣信号：

- 长期只礼貌回复，不主动延续。
- 多次“嗯嗯/哈哈/表情包”结束。
- 用户一直主动，对方很少回到用户身上。
- 拒绝见面且不给替代时间。
- 明确说只当朋友、不想谈、别误会。

反舔狗模式开启时，如果低兴趣信号连续出现，要建议收手，别继续加码。

### 朋友圈和外貌妆容信息

小红书/朋友圈式展示常常比文字更有信息量：

- 妆容、穿搭、滤镜、构图说明对方想展示的审美。
- 评论区互动说明对方社交方式。
- 文案、表情包和微信内置表情说明情绪表达方式。
- 反复出现的场景说明生活偏好。

### 表情包和梗

表情包在中文聊天里经常是语气和情绪补丁。判断时看“这张图在做什么”，不要只看图面可爱不可爱。

- 抽象发疯/奶龙类：常用于装傻、缓冲尴尬、降低暧昧压力、接无厘头。
- 可爱治愈/线条小狗/Chiikawa：常用于委屈、温柔、撒娇、安慰。
- 熊猫头/沙雕文字：常用于自嘲、轻互怼、朋友感。
- 帅气/装酷：常用于被夸后保持松弛和一点反差。
- 长辈图反差：常用于熟人之间故意土味和抽象，早期慎用。

遇到不懂的热梗、IP 或表情包，先查公开中文资料，记录含义、风格、适合场景和雷区。查不到就不要硬用。

这些信息用于选话题、约会地点、礼物和夸法。

### 邀约和约会经验

年轻人约会建议要具体、可执行：

- 520、七夕、情人节、跨年等节点，热门餐厅和花店要提前订。
- 初次见面优先轻量可撤退：咖啡、展览、散步、轻餐，不要一上来昂贵正式。
- 送花/礼物看关系阶段和对方仪式感偏好，暧昧期别砸贵重礼物。
- 西餐不懂就问服务员推荐；点菜先问忌口和预算。
- 旁白提醒和可复制回复必须分开，防止用户误发。

## 回复方案的三类骨架

每次默认给 3 个方案，可按场景替换：

1. 稳妥版：低油腻、低翻车率，适合证据不足或关系早期。
2. 会撩版：有一点暧昧/心动/俏皮，但不得超过阶段油腻上限。
3. 真诚版或降温版：情绪更深或冲突更稳，适合认真话题、磨合和危机。

如果用户要求更进攻的“撩”，先检查：

- 当前阶段是否允许。
- 对方是否给过回应信号。
- 是否会让对方觉得被迫表态。
- 是否符合用户本人风格。
