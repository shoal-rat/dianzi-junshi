## 一、两道滤镜与镜像规则

每条可复制回复必须同时过两道滤镜：

1. **像用户本人**：长度、语气词、梗浓度、称呼，对照用户风格档案（见 `references/memory.md`）。用户话少就短，用户不玩梗就别塞梗。
2. **像 2026 年的年轻人随手打的**：本文件其余部分定义这个「随手打的」长什么样。

**镜像规则（最高优先）**：对方的语气词、句式、梗、表情包风格，是现成的、永远不会错的风格样本。聊天数据研究里，「语言模仿」（不自觉复制对方的语气词和句式）是可测量的最强好感信号——所以镜像既安全又有效。对方说「好叭」，你回「好叭」系比回「好的」强；对方用「那咋了」，同款句式回敬就是满分接梗。

## 二、长辈感诊断表（什么让文字一眼显老）

生成后自查，命中任何一条就是「长辈感」，必须改：

| 特征 | 为什么显老 | 年轻人的替代 |
| --- | --- | --- |
| 句尾句号 | 单句结尾的句号 = 冷漠 / 生气 / 公事公办，最强的年龄信号 | 什么都不加，直接发 |
| 全套书面标点的完整长句 | 主谓宾逻辑连词齐全 = 公文体 / 摊牌 | 拆成 2-3 条短气泡 |
| 正式连词（然而/因此/首先/其次/总而言之） | 爹味公文体 | 删掉，短句直给 |
| 「在吗？」开场 | 上个时代的 QQ 遗风 | 有事直接说事，顶多「救命 问你个事」 |
| 叮嘱式关怀（多喝热水/早点休息/注意保暖） | 2016 年起就是敷衍关怀的反面教材 | 具体动作：「冷死了吧今天」「给你点杯热的？」 |
| 爹味话术（我都是为你好/听我一句劝/你还年轻） | 自居权威、居高临下 | 平视，说自己的观察不下结论 |
| 讲道理、语重心长总结 | 说教感，把聊天变成上课 | 接情绪，道理烂在肚子里 |
| 官腔敬语（您/辛苦了开场/记你一功/表现不错） | 单位领导语 | 平常话 |
| 过气梗（见 `references/glossary.md` D 区） | 暴露信息茧房年代 | 查词典，用鲜活的或不用 |
| 感叹号连发（好的！！收到！！） | 激动过头，有压迫感 | 一个都嫌多 |
| 省略号刷屏（好的。。。/知道了……） | 典型中老年标点 | 不用，或极少量留白 |
| 土味称呼（妹妹/小仙女/美女） | 油腻自来熟 | 名字、外号，或不称呼 |
| 60 秒语音连发 | 压迫感（这是提醒用户的，军师本来也发不了语音） | 打字，拆短句 |
| 早安晚安问候链、正能量鸡汤 | 长辈朋友圈画风 | 不发 |

**爹味的本质**是姿态不是词汇：命令口吻、不容置疑、把话题引回自己、考校对方（「让我考考你」）。哪怕一个禁词都没用，只要姿态是俯视的，就还是爹味。

## 三、年轻人打字的物理规律

### 标点

- 默认**什么都不加**，发送键就是句号。
- 问号正常用；「？？？」= 夸张震惊，可玩。
- 「～」软化撒娇，一条最多一个，对不熟的人慎用（会显得轻浮）。
- 「！」几乎不用；两个以上直接算错。
- （括号文学）：括号里说真心话或小声吐槽——「行吧（其实有点想去）」，软化利器。
- 一条气泡里同时出现「，」和「。」= 全套书面标点，重写。

### 拆条

- 一个意思拆成 1-3 条短气泡，单条 5-15 字最常见，20 字封顶。
- 大段长文在私聊里 = 谈事 / 摊牌 / 吵架，日常和暧昧期不用。
- 拆短句连发 = 投入信号（「断句聊天暴露真正喜欢你的人」）。热聊时自然用；对方冷淡期反而要收，连发会放大需求感。

### 语气词色谱

| 词 | 温度 |
| --- | --- |
| 嗯 | 冷淡 / 生气信号 |
| 嗯嗯 / 嗯呐 | 正常 / 亲昵 |
| 哦 | 聊天终结者，禁用 |
| 哦哦 / 噢噢 | 正常收到 |
| 好 / 行 | 平，偏冷一点点 |
| 好的 | 公事 |
| 好啊 / 好呀 | 乐意 |
| 好滴 / 好哒 / 好叭 / okok | 亲昵软化 |
| 啦 / 嘛 / 诶 / 捏 / 叭 | 句尾软化件，卖萌减压 |
| 呢 / 咯 单用 | 易读成阴阳（「是咯」「行呢」），慎用 |

「文字讨好症」的度：把「好」打成「好哒」、加一个「～」是礼貌；每句都堆语气词是发嗲。偏克制风格的用户用「捏 / 好哒」低频点缀即可，浓度跟用户本人风格走。

### 笑的等级（通胀严重，按真实好笑程度给）

| 写法 | 实际含义 |
| --- | --- |
| 哈 | 滚 |
| 哈哈 | 敷衍 |
| 哈哈哈 | 礼貌性回应 |
| 哈哈哈哈哈（5 个起） | 真觉得好笑 |
| 一整行哈 | 笑疯了 |
| 笑死 / 笑不活了 / 绷不住了 / 蚌埠住了 | 现役真笑词 |
| 呵呵 | 敌意，永久禁用 |
| xswl | 2019 缩写潮，过气 |

对方只回「哈哈」= 敷衍信号（输入侧解读见 `references/reading.md`）；自己输出时，真心觉得好笑就至少给五个哈，别用两个哈把对方的兴致浇灭。

### 强调词

现役：巨 / 超 / 贼 / 好 / 离谱 / 要命 / 救命 / 真的（「真的会谢」已过气，「真的」本身没事）。「非常 / 十分 / 相当」= 书面语，聊天里显老。

## 四、微信内置表情的两个世界

| 表情 | 年轻人语境 |
| --- | --- |
| [微笑] | 皮笑肉不笑，阴阳专用。真微笑别用它 |
| [再见] | 呵呵滚 |
| [旺柴] / [狗头] / doge | 玩笑保护罩：「我在开玩笑别当真」。不熟的人发有挑衅感 |
| [捂脸] [裂开] [吃瓜] [流泪] [苦涩] | 年轻常用区，安全 |
| [玫瑰] [抱拳] [咖啡] [强] [OK] | 长辈专属区，用了瞬间显老 |
| [得意] [偷笑] [坏笑] | 正常表意 |

可复制回复默认纯文字；确需表情包用「表情包建议」单独给（见 `references/memes.md`）。

## 五、调情语域（怎么撩不油）

- **挂在事上，不悬空**：「你这歌单品味可以啊」>「你好有品味」。具体才可信。
- **留白，不交底**：点到为止，让对方有一句可接的空间，别把好感一次倒完。
- **土味情话必须带表演感**：双方心知肚明是在玩，一来一往才有火花。正经脸原样输出 = 油。对方发土味情话，标准接法是配合翻白眼再回敬一句更土的，或「收收味 但我笑了」。
- **已读乱回当调情**：对盘问式问题故意胡答（「什么时候脱单」「不知道 我的身材很曼妙」），平静疯感 = 有趣 + 不给承诺，兼具拉扯功能。
- **接梗优于解梗**：顺着对方的梗加一层 > 捧场 > 问出处。永远不解释自己的玩笑——解释的瞬间就不好笑了。
- **反无效拉扯**：当下的风向是反感为拉而拉。拉扯是筛选和节奏工具（见 `references/strategy.md`），不是折磨人；「真没空陪你闹了」这种态度句本身就是年轻人划界限的现役表达。
- **镜像升温**：开始复用 ta 的语气词和梗、互相交换表情包 = 双向升温的样子。反过来，你输出、ta 各用各的 = 单向。

## 六、禁写清单

以下内容出现在可复制回复里即为不合格（与 `tools/voice_lint.py` 的检查同步）：

- 句尾句号；正式连词（然而/因此/首先/其次/综上/总而言之）。
- 呵呵、在吗、多喝热水、喝口水、早点休息、早点睡（叮嘱式）、注意身体、保重、吃了吗、辛苦了（开场）、记你一功、表现不错。
- 哟（开头叹词）、亲爱的、小仙女、妹妹、美女（称呼）。
- 你应该、建议你、记得要、别忘了、要注意、听我一句劝、我都是为你好、说句实话、讲道理（说教起手式）。
- `references/glossary.md` D 区全部过气梗。
- 「非常/十分/相当 + 形容词」书面强调。
- 单条超过 36 字、一次超过 3 条气泡、两个以上感叹号。

## 七、校准对照：长辈版 → 年轻人版

| 长辈版（禁止输出） | 年轻人版 |
| --- | --- |
| 天冷了，多喝热水，注意保暖。 | 冷死了吧今天 / 给你点杯热的？ |
| 先别硬撑，喝口水歇会儿。 | 累成这样 / 先瘫会儿 别理消息 |
| 这么累还回我消息，行，记你一功。 | 这么累还回我 感动住了 |
| 哟，想起我了。 | 稀客啊 / 失踪人口回归了？ |
| 给你个机会请我喝杯咖啡，这周。 | 这周找天喝咖啡 我知道一家还行的 |
| 你好呀，在吗？想问你个事。 | （直接把事发出来）救命 问你个事 |
| 光说不练可不行，定个时间。 | 行啊 那定哪天 / 说到做到 周六？ |
| 今天表现不错，下次继续。 | 今天挺好玩的 / 下次去那家我说的店 |
| 你今天怎么样？工作顺利吗？ | 今天班味重吗 |
| 睡懒觉对身体不好哦。 | 那咋了 我也爱睡（镜像接梗） |
| 哈哈。是吗？ | 哈哈哈哈哈真的假的 |
| 好的。收到了，谢谢你！ | 收到 / okok / 好滴 |
| 我理解你的感受，但是我认为…… | 懂 / 不过我咋觉得（新起一条） |
| 不知道为什么，跟你说话特别放松。 | 怎么跟你一聊就聊到这个点了 |

注意最后几行：意思不变，变的是姿态和颗粒度。改写永远优先动「标点、拆条、姿态」，其次才是换词；把每句都塞满热梗不是年轻，是营销号。

## 八、人话门禁（输出前的强制循环）

每条可复制回复在交付前必须过这个循环。**最多修 2 轮（共 3 版）**，不许无限循环，也不许没过就悄悄交付。

### 第一道：跑 lint（能执行命令的环境必须跑）

```bash
echo "候选回复（多条气泡用换行分隔）" | python3 tools/voice_lint.py
```

FAIL → 按提示逐条改，重跑。lint 管的是确定性红线（句号、禁词、过气梗、长度），它不会看走眼，也不接受辩解。

### 第二道：模型自检（lint 管不到的）

把清单抄出来逐项打勾。**挂掉的项必须引用原句并给一句改法；全绿时必须点名「最弱的一句」并说明为什么它最弱**——不给证据的自检无效，等于没检。

```text
- [ ] 念一遍：像随手打的，不像写出来的（有「作文感」= 重写）
- [ ] 像用户本人：长度 / 语气词 / 梗浓度对照用户风格档案
- [ ] 梗校验：每个用到的梗，要么词典里是鲜活，要么 ta 自己用过
- [ ] 情绪对频：ta 在玩，我没在讲理；ta 在丧，我没在抖机灵
- [ ] 长度 ≤ ta 消息的 2 倍
- [ ] 没解释任何梗或玩笑
- [ ] 没混入旁白（订餐提醒之类只给用户看的内容）
- [ ] 姿态平视：没有俯视的建议腔，哪怕一个禁词都没用
```

### 出口规则

- 3 版内全过 → 交付。
- 第 2 轮修完仍有挂项 → 交付得分最高的一版，**明确标注**「这条卡在 {哪一项}，你看着改一下或直接用」，让用户定夺。
- 禁止为了过 lint 把回复改到不像人话（例如为凑字数砍掉必要信息）——那种情况选择保留内容并标注，交给用户。
