# 性格底色

在"阶段、兴趣度、聊天习惯"之下，再描一层更慢、更稳的东西——这个人的**性格底色**，以及 ta 的**要 / 不要、喜欢 / 不喜欢**。

底色不是阶段。阶段会变，兴趣会涨会落，但底色是解释"为什么 ta 会这样反应"的那层。把底色描准，回复和约会建议才不只对当下这条消息有效，而是对这个人有效。

底色全部基于证据：聊天里 ta 自己说过的、做过的、反复出现的。证据不足就标 `[待观察]`，别靠星座、MBTI 或第一印象硬填。

读取顺序：先有 `prompts/partner_analyzer.md` 的行为档案，再用本文件把底色补上；要据此设计现场体验时配合 `references/experience_design.md`，要据此调用户气质时配合 `references/persona_modes.md`。

---

## 一、底色的几条轴

每条轴用温度计式判断（偏哪边 + 证据），别贴死标签。一个人可以"对朋友外向、对暧昧对象慢热"，写清楚在什么情境下偏哪边。

| 轴 | 一端 | 另一端 | 怎么看出来 |
| --- | --- | --- | --- |
| 社交能量 | 内向、单线深聊 | 外向、人群里充电 | 朋友圈社交密度、约会偏好热闹还是安静、主动开题频率 |
| 求稳 ↔ 求新 | 喜欢熟悉、可预期 | 喜欢新鲜、爱尝试 | 有没有"想去没去过的地方""想试没试过的事"、对临时变化的反应 |
| 理性 ↔ 情绪 | 先讲逻辑和事 | 先讲感受和氛围 | 倾诉时要建议还是要共鸣、做决定看条件还是看心情 |
| 计划 ↔ 随性 | 提前定、怕意外 | 临时起意、怕被框 | 约会要不要提前敲细节、对"突然约"的接受度 |
| 守规则 ↔ 挑战感 | 习惯按规矩 | 喜欢一点叛逆、反差、打破常规 | 对"乖"和"痞"的反应、聊起出格的事是回避还是来劲 |
| 表达 ↔ 克制 | 有话直说、情绪外放 | 收着、要被读懂 | 是直球还是拐弯、潜台词多不多 |
| 安全感来源 | 掌控感 / 被在乎 / 独立空间 / 被认可 | —— | ta 什么时候最放松，什么时候开始多想 |

"守规则 ↔ 挑战感"这条最值得描准：它决定 ta 喜欢用户走"乖一点"还是"痞一点"的气质（见 `persona_modes.md`）。

## 二、要 / 不要、喜欢 / 不喜欢

把零散证据收成几张具体清单。具体到事，别写"喜欢被关心"这种万能句。

- **喜欢什么**（会让 ta 眼睛亮的）：具体到话题、场景、被对待的方式。例：聊宠物会立刻话变多、被人记住随口提过的小事、安静的展览。
- **不喜欢什么**（会让 ta 兴致掉下来的）：例：被催、被查岗、太吵的局、油腻的夸法、聊前任。
- **想要但没说透的**：ta 表达过、但没明说的需要。例："其实想有人主动安排一下"。
- **想做不敢做**：ta 流露过"想做、但因为习惯/面子/怕麻烦一直没做"的小愿望——想去 livehouse 但没人陪、想试某家店一直没去、想周末出趟远门。这是约会体验设计最好的素材（见 `experience_design.md`）：帮 ta 把"一直想做没做的"自然促成，ta 会觉得和你在一起，有意思的事就这么发生了。只记 ta 自己流露过的，别替 ta 脑补。

## 三、输出格式

在 `/junshi`、`/interest`、`/update-partner` 或需要深判断的 `/reply` 里，必要时输出：

```markdown
## 性格底色：{name}

### 底色轴（偏向 + 证据）
- 社交能量：{内向/外向/分情境}（证据：{一句}）
- 求稳↔求新：{偏稳/偏新/[待观察]}（证据：{一句}）
- 理性↔情绪：{偏理性/偏情绪/[待观察]}（证据：{一句}）
- 计划↔随性：{偏计划/偏随性/[待观察]}（证据：{一句}）
- 守规则↔挑战感：{偏乖/偏痞/分情境/[待观察]}（证据：{一句}）
- 表达↔克制：{直球/收着/[待观察]}（证据：{一句}）
- 安全感来源：{掌控/被在乎/独立空间/被认可/[待观察]}（证据：{一句}）

### 要 / 不要
- 喜欢（眼睛会亮）：{具体清单}
- 不喜欢（兴致会掉）：{具体清单}
- 想要没说透：{方向 + 证据}
- 想做不敢做：{ta 流露过的小愿望 + 证据；没有就写"暂无证据"}

置信度：{高/中/低}
```

## 四、底色怎么用

- **回复**：底色定语气底盘。偏理性的少灌情绪长文，偏情绪的先接住感受；偏挑战感的可以走 persona 里痞一点的版本，偏乖的走稳和暖。
- **约会**：喜欢清单和"想做不敢做"是体验设计的素材库（见 `experience_design.md`）。
- **节奏**：求稳的人别用太猛的拉扯，求新的人别把每次约会都做成一模一样。
- **更新**：底色比阶段稳，但不是不变。出现矛盾的新证据时按 `prompts/memory_engine.md` 的置信度更新，别一条消息就推翻整层底色。
