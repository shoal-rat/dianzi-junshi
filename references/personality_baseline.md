# 性格底色与边界

这个文件解决一件事：在"阶段、兴趣度、聊天习惯"之下，再描一层更慢、更稳的东西——这个人的**性格底色**，以及 ta 的**要 / 不要、喜欢 / 不喜欢、能碰 / 不能碰**。

底色不是阶段。阶段会变，兴趣会涨会落，但底色是解释"为什么 ta 会这样反应"的那层。把底色描准，回复和约会建议才不会只对当下这条消息有效，而是对这个人有效。

底色全部基于证据：聊天里 ta 自己说过的、做过的、反复出现的。证据不足就标 `[待观察]`，不要靠星座、MBTI 或第一印象硬填。

读取顺序：先有 `prompts/partner_analyzer.md` 的行为档案，再用本文件把底色和边界补上；要据此设计现场体验时，配合 `references/experience_design.md`；要据此调用户人设时，配合 `references/persona_modes.md`。

---

## 一、底色的几条轴

每条轴用温度计式判断（偏哪边 + 证据），不要贴死标签。一个人可以"对朋友外向、对暧昧对象慢热"，写清楚在什么情境下偏哪边。

| 轴 | 一端 | 另一端 | 怎么看出来 |
| --- | --- | --- | --- |
| 社交能量 | 内向、单线深聊 | 外向、人群里充电 | 朋友圈社交密度、约会偏好热闹还是安静、聊天主动开题频率 |
| 求稳 ↔ 求新 | 喜欢熟悉、可预期 | 喜欢新鲜、爱尝试 | 有没有"想去没去过的地方""想试没试过的事"、对临时变化的反应 |
| 理性 ↔ 情绪 | 先讲逻辑和事 | 先讲感受和氛围 | 倾诉时要建议还是要共鸣、做决定看条件还是看心情 |
| 计划 ↔ 随性 | 提前定、怕意外 | 临时起意、怕被框 | 约会要不要提前敲定细节、对"突然约"的接受度 |
| 守规则 ↔ 挑战感 | 习惯按规矩、怕越界 | 喜欢一点叛逆、反差、打破常规 | 对"乖"和"痞"的反应、聊起出格的事是回避还是来劲（直接关系到 persona 偏好） |
| 表达 ↔ 克制 | 有话直说、情绪外放 | 收着、要被读懂 | 是直球还是拐弯、潜台词多不多 |
| 安全感来源 | 掌控感 / 被在乎 / 独立空间 / 被认可 | —— | ta 什么时候最放松，什么时候开始多想 |

"守规则 ↔ 挑战感"这条轴最值得描准：它既决定 ta 喜欢用户走"乖一点"还是"痞一点"的人设（见 `persona_modes.md`），也决定哪些舒适圈边缘可以轻轻碰、哪些绝对不能碰（见下面第三节）。

## 二、要 / 不要、喜欢 / 不喜欢（软肋与兴奋点）

把零散证据收成四张具体的清单。具体到事，不要写"喜欢被关心"这种万能句。

- **喜欢什么**（会让 ta 眼睛亮的）：具体到话题、场景、被对待的方式。例：聊宠物会立刻话变多、被人记住随口提过的小事、安静的展览。
- **不喜欢什么**（会让 ta 兴致掉下来的）：例：被催、被查岗、太吵的局、油腻的夸法、聊前任。
- **想要但没说透的**：ta 表达过、但没明说的需要。例："其实想有人主动安排一下"。当成需要被满足的方向，不是被利用的把柄。
- **嘴上不要、心里可能要的**：走 `partner_analyzer.md` 的潜台词字典，但只在有 ta 个人证据时才这么解读，别套通用模板。

### 想做不敢做（latent wish）

有些人会露出"想做、但因为习惯/面子/怕麻烦没做"的小愿望——想去 livehouse 但没人陪、想试某家店一直没去、想周末出趟远门。

- 只在 **ta 自己流露过** 时才记，不要替 ta 脑补愿望。
- 它的用途是：设计一个 ta 本来就会喜欢、会发光的共同体验（见 `experience_design.md`），让 ta 觉得"和你在一起会发生有意思的事"。
- 它**不是**用来推 ta 去做不想做的事的理由。"ta 没说想要"和"ta 说过不要"都按"不要"处理。

## 三、边界地图（这一节是硬规则，优先级高于任何技巧）

把 ta 的边界分成两类，分错了就会从"会撩"变成"让人不舒服"。分不清时，一律当硬边界。

### 硬边界 / 绝对雷区——永不推、永不"测试"、永不当成要攻克的障碍

包括但不限于：

- ta 明确说过的"不行 / 不想 / 别"，以及任何形式的拒绝和回避。
- 安全、身体、亲密方面的任何分寸——只跟随 ta 主动、清醒、明确的意愿，任何犹豫都按"不要"处理。
- 工作、学业、家庭、健康、作息、钱、宗教信仰这些 ta 的真实责任和底线。
- ta 的隐私、行踪、身材、收入，以及任何 ta 表现出敏感的话题。

对硬边界，正确的动作是**尊重和让路**，不是设计话术绕过去。一个会因为被推过界而"心动"的剧本不存在；只有事后觉得被冒犯的人。

### 软边界 / 舒适圈边缘——只在 ta 真的来劲时，轻轻邀请，一犹豫就立刻收

这是"习惯"而非"原则"的部分：平时习惯早回家、很少吃辣、没怎么试过某类活动、不太主动拍照。这些是偏好，不是底线。

- 可以**邀请**式地轻轻拓展："要不要顺路去看看那个，就十分钟"——给一个低成本、可随时退出的选项。
- 判断 ta 是**真的想**还是**在迁就你**：眼睛亮、自己加细节、主动延续 = 真的想；客气地"都行 / 那好吧"、节奏变迟疑 = 在迁就。拿不准就当迁就，停。
- 一旦出现任何犹豫、找借口、变安静、看时间，立刻把台阶递回去："时间也不早了，今天就到这，下次再说。" 让 ta 走得体面，比多留十分钟值钱得多。

### 怎么区分硬 / 软

- ta **说过**的、关于安全/身体/责任/价值观的 → 硬。
- 只是**习惯和节奏**、ta 没表过态的 → 软，但仍然只在 ta 主动来劲时碰。
- 分不清 → 当硬。宁可错过一次"再坐十分钟"，也不要换来一次"这人没分寸"。

## 四、输出格式

在 `/junshi`、`/interest`、`/update-partner` 或需要深判断的 `/reply` 里，必要时输出：

```markdown
## 性格底色：{name}

### 底色轴（偏向 + 证据）
- 社交能量：{内向/外向/分情境}（证据：{一句}）
- 求稳↔求新：{偏稳/偏新/[待观察]}（证据：{一句}）
- 理性↔情绪：{偏理性/偏情绪/[待观察]}（证据：{一句}）
- 计划↔随性：{偏计划/偏随性/[待观察]}（证据：{一句}）
- 守规则↔挑战感：{偏乖/偏痞/分情境/[待观察]}（证据：{一句}）
- 表达↔克制：{直球/收着/[待观察]}（证据：{一句}）
- 安全感来源：{掌控/被在乎/独立空间/被认可/[待观察]}（证据：{一句}）

### 要 / 不要
- 喜欢（眼睛会亮）：{具体清单}
- 不喜欢（兴致会掉）：{具体清单}
- 想要没说透：{方向 + 证据}
- 想做不敢做：{ta 自己流露过的小愿望 + 证据；没有就写"暂无证据"}

### 边界地图
- 硬边界 / 绝对雷区：{清单}——尊重、让路，不设计话术绕过
- 软边界 / 舒适圈边缘：{清单}——只在 ta 主动来劲时轻邀请，一犹豫就收
- 不确定归类：{清单}——按硬边界处理

置信度：{高/中/低}
```

## 五、底色怎么用

- **回复**：底色决定语气底盘。偏理性的少灌情绪长文，偏情绪的先接住感受；偏挑战感的可以走 persona 里痞一点的版本，偏乖的走稳和暖。
- **约会**：喜欢清单和"想做不敢做"是体验设计的素材库；边界地图是体验设计的红线（见 `experience_design.md`）。
- **节奏**：求稳的人别用太猛的拉扯，求新的人别把每次约会都做成一模一样。
- **更新**：底色比阶段稳，但不是不变。出现和已记底色矛盾的新证据时，按 `prompts/memory_engine.md` 的置信度更新，不要一条消息就推翻整层底色。

## 边界优先级声明

本文件描述的所有"喜欢、想要、舒适圈拓展"都服从一条总规则：**人是用来一起开心的，不是用来攻克的。** 任何让对方事后觉得被算计、被越界、被消费的设计，都判为失败，不管它当下看起来多"有效"。这条优先级高于本仓库里任何拉扯、推拉、人设技巧。
