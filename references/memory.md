档案就是长期记忆，反馈让它长大。判断规则见 `references/reading.md`，策略打法见 `references/strategy.md`，建档模板见 `references/partner.md`。

## 一、原则与启动

### 档案即记忆

- `partners/{slug}/profile.md` 就是长期记忆本身。不是静态文档——每次有效反馈都更新它。
- **跨窗口连续性**：只要 profile.md 在，任何新窗口都能恢复全部学到的知识。用户不需要重新解释"上次说过 ta 不喜欢长回复"。文件就是记忆。
- **各存各的**：每个对象一个 `partners/{slug}/`，阶段、记忆、统计互不串戏。新建对象=空文件夹=全新记忆；删文件夹=清空该对象记忆。绝不读别人的档案。

### 会话启动协议

新窗口第一次处理任何命令（/reply、/ask、/analyze……）时：

**步骤 1 加载档案**——读 profile.md 的：关系概览（阶段、基本信息）、沟通风格和潜台词字典、兴趣度档案、朋友圈画像、微信息记忆、梗记忆、拉扯/节奏策略记忆、海王/玩家信号记录、追法类型记录、学习记忆层、用户偏好记忆；另加最新 `versions/analysis_vN.md`，以及与当前问题相关的 `materials/image_observations.jsonl` 记录。

**步骤 2 激活置信度**：

- `●●●`（3+ 次验证）→ **强制应用**，硬性约束
- `●●○`（2 次验证）→ **优先参考**，强建议
- `●○○`（1 次观察）→ **弱参考**，记录但不主导

**步骤 3 输出恢复摘要**（简短，不打扰）：

```text
已加载 {name} 的记忆档案
  当前阶段：{stage} · 油腻上限 {N}/5
  已验证策略：{N} 条 ●●●  {N} 条 ●●○
  上次对话：{日期 / 首次对话}
{最关键的一条高置信度记忆，如："记得：ta 疲惫时最吃「被看穿」型回复"}
```

## 二、记忆法则

### 先记事件，再升规则

核心纪律：**规则从事件里长出来，不从单次感觉里长出来。**

1. **每次反馈先记事件（EPISODE）**：日期 + 实际发送原文 + 对方回了什么 + 隔了多久 + 热度变化。事件是账本，不做解读。
2. **假设（HYPOTHESIS）从 ●○○ 开始**：从事件里提出"ta 可能吃调侃"这类假设，标 ●○○。
3. **相似场景 2-3 次一致，才升 ●●● 成为常备规则**。一次逗笑 ≠ "ta 永远吃调侃"。场景不同的成功不算同一模式的验证。
4. **负面结果权重更高**：一次聊死的消息，立刻进无效模式表，原句照抄，列入「再也不这样」清单，不用等第二次。
5. **中性结果不写**。情境因素太多，写了只会污染档案。
6. **唯一例外**：用户明确纠正=最高可信来源，直接 ●●●，跳过阶梯（见第五节）。

### 置信度阶梯

```text
初次观察 → ●○○（弱参考）
同一模式第 2 次确认 → ●●○（优先参考）
同一模式第 3 次以上确认 → ●●●（强制应用）
同一模式 = 相似场景 + 相似策略 + 相似结果
```

### 冲突处理

- `●●●` 被推翻 → 降级 `●○○`，加注 `[冲突，待观察]`。
- `●○○` 被推翻 → 直接移入无效模式表。
- 可能是情境差异 → 拆成两条，各标适用条件。例：「日常闲聊用幽默 ●●●」被"ta 情绪低时幽默没回"冲击 → 细化为「情绪正常时幽默有效 ●●●」+ 新增「ta 情绪低时幽默无效 ●○○」。

### 记忆老化与写入格式

- 超过 3 个月未验证的 `●○○` → 标 `[待验证]`，不再主动应用。`●●○`/`●●●` 不老化，除非用户否定。
- 每次写入都追加，不删旧记录（只更新置信度和最近验证日期）。日期粒度 `YYYY-MM`。

### 记忆压缩（上下文纪律）

档案只增不减会越来越大，每次会话都整份读进上下文，读得越多噪声越多。所以档案要定期压缩——原则是**压缩过程、保留结论、归档原始**：

**触发**（任一满足就做）：profile.md 超过约 400 行；学习记忆层事件条目超过 30 条；距上次压缩超过 3 个月。

**流程**：

1. `●●●` 规则和「再也不这样」清单**原样保留**——这是决策层，一个字不动。
2. 已升级成规则的同场景事件组 → 折叠成一行摘要：`「2026-03~05 共 5 次验证，原始记录见 history/」`。
3. 标了 `[待验证]` 超过 6 个月仍无新证据的 `●○○` → 移出工作档案。
4. 被折叠、被移出的条目**不删除**：整段移入 `partners/{slug}/history/archive_{YYYY-MM}.md`。原始账本永远在，只是不再每次进上下文。
5. 压缩完在档案概览记一行：`上次压缩：{日期}，折叠 {N} 条，归档 {N} 条`。

**读取纪律**（每次会话都适用，不只压缩时）：只加载当前对象的档案；image_observations.jsonl 只读与当前问题相关的 image_id；多对象绝不同时加载。需要翻旧账时再去 history/ 里查，别默认背着全部历史聊天。

## 四、写入与读取

### 写入触发表

| 事件 | 写入位置 |
|------|---------|
| 反馈效果"非常好/好"（口头或聊天记录） | 有效策略表，置信度 +1 次 |
| 反馈"较差/没效果"、ta 没回、聊死 | 无效模式表（原句照抄），一次即入 |
| 反馈"中性" | 不写 |
| 用户纠正"ta 不是这样的" | 真实模式修正，直接 ●●● |
| 用户实际选择/修改某方案 | 用户偏好记忆 +1 计数 |
| 确认关系、见面、重要日期 | 关系进展日志 / 概览重要日子 |
| 输出四维兴趣判断 | 兴趣度变化记录 + meta.json |
| 用了策略框且有结果 | 拉扯/节奏策略记录 |
| 判断海王置信度 / 命中玩家信号 / 追法类型 | 对应三张记录表 |
| 上传朋友圈截图 / 导入图片 | 朋友圈画像 / image_observations.jsonl |
| 提取到忌口、口头禅、MBTI 等 | 微信息记录 |
| ta 用了梗（新的或已知的） | 梗记忆；全局新梗另写 glossary.md |
| 大量导入 / 阶段或兴趣重估 | versions/analysis_vN.md |
| 开关反舔狗模式 | meta.json + 概览 |

### 读取优先级

```text
1. ●●● 有效策略 → 同场景优先推荐
2. ●●● 无效模式 → 同场景绝对不给
3. ●●○ 有效策略 → 优先参考
4. ●●○ 无效模式 → 尽量避免
5. ●○○ → 弱参考，不主导
```

### 应用规则

- **用户偏好**：某类策略选择率 >50% 就放第一位；有固定修改习惯（如总删感叹号）就生成时直接照做。
- **兴趣度记忆**：先看四维再看总分；甜度高主动低→少追问改展示自己；行动兑现低→先行动测试，不堆情绪价值。判断阈值和反舔狗硬阈值一律以 `references/reading.md` 为准，本文件不复述。
- **节奏记忆**：优先用对这个对象验证过有效的回复间隔；停太久冷场就缩短，秒回后降温就下次拉长。动作定义见 `references/strategy.md`。
- **海王/玩家信号记忆**：分数升→减少投入、优先测试动作；分数降→按四维继续推进。
- **追法类型记忆**：已验证有效的拉扯强度优先；无效就降一级或换打法（见 `references/strategy.md`）。
- **梗记忆**：可复制回复默认纯文字；ta 在用的梗可镜像；没查证的梗宁可不用（见 `references/memes.md`）。

## 五、反馈闭环

### 触发

`/upload-followup`、"发完之后的对话是这样的"、"ta 的反应是…"（截图/文字/WeChatMsg 导出都收）。

### 流程

1. **提取上次建议**：用户选了哪个方案（或自改版本）、策略类型、策略框里的间隔/停顿/动作/观察点。
2. **解析后续记录**：用户实际发送原文、ta 回了什么、隔多久、互动轮次、是否反问/补细节/给替代时间、是否按建议停顿、停顿后 ta 补没补主动。
3. **效果分级**：

| 等级 | 标准 |
|------|------|
| 非常好 | ta 情绪明显变好，话变多，主动延续 |
| 好 | 正常回应，对话继续，氛围好 |
| 中性 | 回了，但没推进（不写入记忆） |
| 较差 | 简短冷淡，或转移话题 |
| 没效果 | 没回，或情绪变差 |

4. **归因**：策略加分点（哪个词起了作用）/ 策略风险点 / 节奏因素（间隔、停顿）/ 情境因素（ta 当时在忙，与策略无关）。
5. **跑 `tools/session_log.py`** 记录 session。字段：session_id、message_analyzed、option_used、strategy、reply_interval_advice、actual_reply_interval、pause_advice、pushpull_action、pursuit_type、pushpull_intensity、recent_player_signal、player_test_action、actual_reply、partner_response、follow_up_turns、outcome、interest_delta、player_confidence_delta、attribution、notes。
6. **按第二节记忆法则写入 profile.md**：先记事件，效果好→有效策略表升置信度，聊死→无效模式表带原句，中性不写。同步更新兴趣度、节奏、海王、用户偏好各表。

### 写入确认输出

```text
已存进档案。
  记忆更新：{有效策略表 / 无效模式表} · 场景：{描述}
  置信度：{●○○ 首次 / ●●○ 第 2 次 / ●●● 第 3 次+，已成强规则}
```

### 轻量口头反馈

不上传记录也记，置信度标"主观报告"，权重低于解析数据：

- "效果很好" → outcome: 好　"一直在聊" → 非常好
- "ta 没回" → 没效果　"回了但冷冷的" → 较差

### /stats

某对象累计 5 条以上效果记录后可输出：哪类策略最有效、哪类表达让 ta 冷淡、最佳发消息时段、ta 接不接"会撩版"、哪种回复间隔有效、停顿后 ta 补不补主动、海王置信度趋势、哪个追法类型和拉扯强度最有效。

### 用户纠正（最高优先级写入）

触发："这不像 ta"、"ta 不是这样的"、"这个方向不对"、"我不会这样说话"。

1. 确认："我理解了，你是说 {name} 不是{原描述}，而是{新描述}，对吗？"
2. 写入真实模式修正表，**直接 ●●●**——用户纠正是最高可信来源，跳过 2-3 次事件阶梯（这是记忆法则的唯一例外）。
3. 同时修改档案原字段，旁注 `（已纠正，见记忆层）`。**不删原记录**，原描述标 `[已纠正]` 保留历史。
4. 策略被否定（"这类建议对 ta 没用"）→ 无效模式表，●●●。
5. 立即生效，回复附确认：

```text
好的，记住了，下次会调。
  已更新：{维度}："{原}" → "{新}"　置信度：●●●
  这条在所有新窗口生效。
```

不质疑用户的纠正——他们最了解自己的对象。

## 六、用户风格校准、分析版本、/memory

### 两重过滤

每条建议要过两关：像不像**对象**合适的话（对象档案，见 `references/partner.md`）+ 像不像**用户本人**会说的话（本节）。

### 用户风格采集

- **口述**：从"话少/喜欢撩/比较正经/常用 hh/不用表情包"等自述映射成行为规则（话少→1-3 句不展开；常用 hh→用 hh 替代哈哈哈）。
- **聊天记录推断**（更可信）：消息密度、典型长度、语气词（嗯嗯/好的/hh）、标点习惯（加不加句号、感叹号频率）、表情包类型和频率、对 ta 的称呼、幽默方式。

风格档案写进 profile.md：消息格式（密度/长度/真实例句）、语言习惯（语气词/标点/表情包/称呼）、语气倾向（总体风格/撩的程度/幽默方式/展示自己方式）。

### 应用规则

- 语气词、长度、标点、称呼全部对齐用户习惯；用户话少建议就短。
- 用户从不用表情包就不推荐；可复制回复默认纯文字。
- 用户正经就不给过暧昧方案；梗要用户会说的，不确定先查。
- 展示自己按用户能力包装：话少→一句状态+一个轻问题；不会展示→给能直接发的短句。
- 机器腔过滤：删掉用户不会说的整齐表达、解释腔、万能总结（门禁见 `references/voice.md`）。
- **从修改中学习**：用户每次改动都进"实际修改记录"，形成选择率统计（见第三节用户偏好记忆）。
- 痞/乖气质放大：在用户真实底盘上调，读 `references/strategy.md`。
- 无信息时默认：一次 2-3 条、每条 10-25 字、自然有温度、不配表情包、轻微撩。

### 分析结论版本（versions/analysis_vN.md）

**何时生成**：完整大复盘、阶段重估、兴趣重估、批量导入材料后。它是可回看的证据账本——让用户以后知道当时为什么判暧昧/单向上头/止损。

模板要点（每节 1-3 行）：

```markdown
# analysis_v{N} - {name}　生成：{日期}　材料范围：{…}　上一版本：{vN-1/无}
## 一句话结论
## 先破题名：用户标签 vs 证据支持的真实阶段 + 不采信原因
## 四维兴趣分：四行分数+证据，总分+置信度
## 关键证据：加分/减分/结论性（能回指原文或图片 ID）
## 图片证据索引：img_id | 支持什么结论 | 关键可见证据
## 阶段判断：当前阶段 + 为什么不是更高 + 升级需要什么证据
## 反舔狗判断：是否触发硬阈值（阈值见 references/reading.md）+ 建议
## 下一步策略：先做 / 不做 / 验证点
## 待纠错点：低置信度结论，等用户补材料
```
