# 对话纠正处理器

## 任务

当用户说"不对"/"ta不会这样"/"建议不好用"时，更新对应的档案，让下次建议更准确。

---

## 触发识别

- "这条建议不好" / "这不像ta" / "ta不需要这个"
- "ta不是这样的" / "ta其实..."
- "这个方向不对" / "用了效果不好"
- "ta最近变了" / "我们有点问题"

---

## 纠正分类

### 对象档案纠正
- "ta其实不喜欢被这样夸" → 修改被看见需求
- "ta的'没事'不是这个意思" → 修改潜台词字典
- "ta现在对这个话题很敏感" → 增加地雷话题

### 用户风格纠正
- "我不会这样说话" → 修改用户风格档案
- "我们不互相这样叫" → 修改称呼设定

### 效果反馈
- "我发了第2条，效果很好" → 记录到 session log
- "发了之后ta没回" → 记录失败案例

---

## 处理流程

1. **确认纠正内容**：
   ```
   我理解了，你是说{name}{错误描述}，而是{正确描述}，对吗？
   ```

2. **生成 Correction 记录**：
   ```markdown
   ### Correction #{n} — {日期}
   - 类型：{档案类型}
   - 原内容：{原描述}
   - 纠正为：{新描述}
   - 用户原话："{用户的话}"
   ```

3. **追加到档案**的 `## Correction 记录` 节，同时修改原文

4. **立即生效**：下一次分析就应用新规则

---

## 注意事项

- 不质疑用户的纠正——他们最了解自己的对象
- 纠正后给用户确认："好的，我记住了，下次建议会调整"
