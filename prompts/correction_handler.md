# 对话纠正处理器

## 任务

当用户说"不对"/"ta不会这样"/"建议不好用"时，更新对应的档案，让下次建议更准确。

---

## 触发识别

- "这条建议不好" / "这不像ta" / "ta不需要这个"
- "ta不是这样的" / "ta其实..."
- "这个方向不对" / "用了效果不好"
- "ta最近变了" / "我们有点问题"

---

## 纠正分类

### 对象档案纠正
- "ta其实不喜欢被这样夸" → 修改被看见需求
- "ta的'没事'不是这个意思" → 修改潜台词字典
- "ta现在对这个话题很敏感" → 增加地雷话题

### 用户风格纠正
- "我不会这样说话" → 修改用户风格档案
- "我们不互相这样叫" → 修改称呼设定

### 效果反馈
- "我发了第2条，效果很好" → 记录到 session log
- "发了之后ta没回" → 记录失败案例

---

## 处理流程

1. **确认纠正内容**：
   ```
   我理解了，你是说{name}{错误描述}，而是{正确描述}，对吗？
   ```

2. **生成 Correction 记录**：
   ```markdown
   ### Correction #{n} — {日期}
   - 类型：{档案类型}
   - 原内容：{原描述}
   - 纠正为：{新描述}
   - 用户原话："{用户的话}"
   ```

3. **追加到档案**的 `## Correction 记录` 节，同时修改原文

4. **立即生效**：下一次分析就应用新规则

---

## 步骤 4：写入长期记忆

**纠正信息比反馈更重要——它直接修正了原来的认知。**

按照 `prompts/memory_engine.md` 的规则处理：

### 4.1 对象档案纠正 → 写入真实模式修正表

```markdown
| {纠正维度} | {原始描述} | {用户纠正后的描述} | ●●● | 用户纠正 |
```

置信度直接设为 `●●●`（用户主动纠正 = 最高可信度来源）。

同时修改档案中对应字段的原始描述，并在旁边标注 `（已纠正，见记忆层）`。

### 4.2 策略效果纠正 → 写入无效模式表

如果用户说"这类建议对ta没用"：

```markdown
| {场景描述} | {无效策略类型} | {纠正原话} | ●●● | 用户直接否定 |
```

### 4.3 纠正不会删除原记录

原始描述保留但标注 `[已纠正]`，确保保留修改历史。

### 4.4 写入确认

在回复用户时附加：

```
好的，我记住了，下次建议会调整。

已更新记忆：
  {纠正维度}："{原描述}" → "{新描述}"
  置信度：●●●（直接更正，最高优先级）
  
  这条记忆会在所有新窗口中生效。
```

---

## 注意事项

- 不质疑用户的纠正——他们最了解自己的对象
- 纠正后立即写入记忆层，不只是当次生效
