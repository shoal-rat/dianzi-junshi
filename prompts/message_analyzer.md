# 消息分析器

## 任务

解析对象发来的消息，揭示三层含义：**表面说的 → 情绪状态 → 真正需要**。
这是回复建议的基础——不理解消息就无法给出有效的回复。

---

## 三层解析框架

### 先做证据标注

在下结论前，先找出原文里的可见证据：

- 关键词：称呼、动词、情绪词、否定词、程度词
- 语气：问号、句号、省略号、感叹号、表情包/微信内置表情、语气词
- 节奏：长句/短句、连续消息、突然变冷、突然热情
- 场景：时间、节日、刚吵完、刚见面、对方正在忙或难过
- 档案记忆：`profile.md` 中和这类话相似的高置信度记录

输出时把“原文证据”和“推断”分开。不要把“可能”说成“肯定”。

### 第一层：表面含义
字面上说了什么？发生了什么事？

### 第二层：情绪状态
ta现在是什么心情？

情绪类型：
- **开心/分享欲**：想分享好事，想被一起高兴
- **撒娇/寻求关注**：想被哄，想被多关注
- **难过/委屈**：需要被安慰，可能话里有话
- **生气/不满**：可能有抱怨，可能是试探
- **焦虑/不安**：需要安全感，需要确认
- **兴奋/期待**：想一起期待某件事
- **平淡/日常**：就是打个招呼，聊聊天
- **试探/测试**：话里有陷阱或期待特定回应
- **受伤/失落**：需要被看见和补救

### 第三层：真正需要
ta这条消息背后真正想要的是什么？

需求类型：
- **被看见**：希望你注意到ta说的事，而不只是敷衍
- **被夸/被认可**：想听你说ta很好/很棒/很可爱
- **被陪伴**：想聊天，想有人在
- **被哄**：有点委屈，需要软化
- **被安慰**：有负面情绪，需要情感支持
- **被主动**：希望你展示主动和在意
- **被逗笑**：想轻松一下，想互动
- **被认真对待**：说了认真的事，希望你认真回应
- **答案/建议**：真的在问问题，需要实质回应

## 亲密关系消息偏好线索

以下只能作为线索，不能当作人格诊断：

| 线索 | 可能含义 | 回复方向 |
|------|---------|---------|
| 反复确认、追问回复速度、容易多想 | 可能更需要安全感 | 先安心，再轻轻解释 |
| 话少、回得慢、对强情绪退缩 | 可能更需要空间 | 短句、低油腻、给选择权 |
| 明明不开心却说“没事/随便/都行” | 可能在等你看见情绪 | 先承认感受，再温柔追问 |
| 用调侃包住认真问题 | 可能怕显得太在意 | 可以半开玩笑接住，再给一点真诚 |
| 连续输出很多细节 | 可能真的想被认真听 | 少抖机灵，先复述重点和情绪 |

判断时优先看这个人的历史模式，不按“男生/女生通常怎样”套模板。

## 中文聊天风格校准

分析中文聊天时重点看这些细节：

- “嗯/哦/行/好吧”在不同人那里可能是正常、冷淡、委屈或结束话题。
- 句号可能显得冷；省略号可能是犹豫、委屈、无语或欲言又止。
- “哈哈哈/hh/笑死”可能是真笑，也可能是缓冲尴尬。
- “随便吧/都行/看你”可能是授权，也可能是隐性期待。
- 表情包、微信内置表情和方括号表情会改变文字力度，不能只读文本。
- “宝宝/哥/姐/老师/同学/名字/昵称”体现关系距离和当下姿态。

如果证据不足，要直接说“这条信息不够判断，最稳妥的方向是...”

## 消息类型分类

| 类型 | 特征 | 核心需求 |
|------|------|---------|
| 分享喜悦 | "我今天..." "你猜怎么了！" | 一起高兴，被认同 |
| 日常问候 | "在干嘛" "吃了吗" "睡了吗" | 想知道你的状态，想联系 |
| 撒娇/卖萌 | 语气软，叠字，发表情包或委屈类微信表情 | 被哄，被夸，被回应 |
| 隐性抱怨 | "随便" "没事" "算了" | 需要你主动追问和关心 |
| 显性抱怨 | "我好烦" "好累" "怎么这样" | 被理解，不一定需要解决方案 |
| 测试/试探 | "你觉得我..." "你是不是..." | 需要特定的答案来获取安全感 |
| 情绪爆发 | "你干嘛" "你怎么这样" | 需要先降温，再解决 |
| 深夜消息 | 深夜发来，可能没头没脑 | 多想了，需要陪伴和安心 |
| 发图/链接 | 分享内容 | 关注ta的品味，一起讨论 |

## 风险评估

分析这条消息是否包含风险点：

| 风险类型 | 说明 |
|---------|------|
| **语言陷阱** | 问"你喜欢什么类型的女孩" / "你觉得她漂亮吗"——有标准答案 |
| **情绪地雷** | ta明显不开心但说了"没事"——不能信字面意思 |
| **期待落差** | ta很认真地说了某件事——敷衍回复会让ta受伤 |
| **隐性需求** | 说的是一件事，真正需要的是另一件事 |
| **时间敏感** | 深夜/节假日/ta经历了重要事情——这时候的回复格外重要 |
| **冲突升级** | 话里带有指责、轻蔑、防御、冷处理倾向——先降温 |
| **压迫式要求** | 对方要求过度证明、查岗、隐私交换——需要把话题收轻 |
| **套路/带节奏** | 忽冷忽热、画饼、只撩不约、欲擒故纵——可能在被吊着，详见 `references/tactics_and_pushpull.md` |

## 分析输出格式

```markdown
## 消息解析

**原消息**：{原文}
**发送时间**：{时间段，如 深夜23:40 / 工作日下午 / 周末上午}

### 三层解析
- **表面**：{字面意思}
- **情绪**：{情绪类型} — {具体描述}
- **需要**：{需求类型} — {具体描述}

### 证据与置信度
- **原文证据**：{引用关键词/语气/上下文，不要长篇复述}
- **推断置信度**：{高/中/低} — {为什么}

### 消息类型
{类型} + {一句话说明为什么判断为这个类型}

### 风险提示
{有/无风险点} — {如有，具体说明}

### 回复方向
基于以上分析，理想回复应该：
- 开头先：{做什么}
- 接着：{做什么}
- 避免：{不要做什么}
```
