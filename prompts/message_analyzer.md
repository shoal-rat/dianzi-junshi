# 消息分析器

## 任务

解析对象发来的消息，揭示三层含义：**表面说的 → 情绪状态 → 真正需要**。
这是回复建议的基础——不理解消息就无法给出有效的回复。

---

## 三层解析框架

### 第一层：表面含义
字面上说了什么？发生了什么事？

### 第二层：情绪状态
ta现在是什么心情？

情绪类型：
- 😊 **开心/分享欲**：想分享好事，想被一起高兴
- 🥺 **撒娇/寻求关注**：想被哄，想被多关注
- 😔 **难过/委屈**：需要被安慰，可能话里有话
- 😤 **生气/不满**：可能有抱怨，可能是试探
- 😰 **焦虑/不安**：需要安全感，需要确认
- 🤩 **兴奋/期待**：想一起期待某件事
- 😐 **平淡/日常**：就是打个招呼，聊聊天
- 🤔 **试探/测试**：话里有陷阱或期待特定回应
- 💔 **受伤/失落**：需要被看见和补救

### 第三层：真正需要
ta这条消息背后真正想要的是什么？

需求类型：
- **被看见**：希望你注意到ta说的事，而不只是敷衍
- **被夸/被认可**：想听你说ta很好/很棒/很可爱
- **被陪伴**：想聊天，想有人在
- **被哄**：有点委屈，需要软化
- **被安慰**：有负面情绪，需要情感支持
- **被主动**：希望你展示主动和在意
- **被逗笑**：想轻松一下，想互动
- **被认真对待**：说了认真的事，希望你认真回应
- **答案/建议**：真的在问问题，需要实质回应

---

## 消息类型分类

| 类型 | 特征 | 核心需求 |
|------|------|---------|
| 分享喜悦 | "我今天..." "你猜怎么了！" | 一起高兴，被认同 |
| 日常问候 | "在干嘛" "吃了吗" "睡了吗" | 想知道你的状态，想联系 |
| 撒娇/卖萌 | 语气软，叠字，发表情 | 被哄，被夸，被回应 |
| 隐性抱怨 | "随便" "没事" "算了" | 需要你主动追问和关心 |
| 显性抱怨 | "我好烦" "好累" "怎么这样" | 被理解，不一定需要解决方案 |
| 测试/试探 | "你觉得我..." "你是不是..." | 需要特定的答案来获取安全感 |
| 情绪爆发 | "你干嘛" "你怎么这样" | 需要先降温，再解决 |
| 深夜消息 | 深夜发来，可能没头没脑 | 多想了，需要陪伴和安心 |
| 发图/链接 | 分享内容 | 关注ta的品味，一起讨论 |

---

## 风险评估

分析这条消息是否包含风险点：

| 风险类型 | 说明 |
|---------|------|
| **语言陷阱** | 问"你喜欢什么类型的女孩" / "你觉得她漂亮吗"——有标准答案 |
| **情绪地雷** | ta明显不开心但说了"没事"——不能信字面意思 |
| **期待落差** | ta很认真地说了某件事——敷衍回复会让ta受伤 |
| **隐性需求** | 说的是一件事，真正需要的是另一件事 |
| **时间敏感** | 深夜/节假日/ta经历了重要事情——这时候的回复格外重要 |

---

## 分析输出格式

```markdown
## 消息解析

**原消息**：{原文}
**发送时间**：{时间段，如 深夜23:40 / 工作日下午 / 周末上午}

### 三层解析
- **表面**：{字面意思}
- **情绪**：{情绪类型} — {具体描述}
- **需要**：{需求类型} — {具体描述}

### 消息类型
{类型} + {一句话说明为什么判断为这个类型}

### 风险提示
{有/无风险点} — {如有，具体说明}

### 回复方向
基于以上分析，理想回复应该：
- 首先：{做什么}
- 其次：{做什么}
- 避免：{不要做什么}
```
