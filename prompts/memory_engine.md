# 长期记忆引擎

## 设计原则

档案文件（`partners/{slug}/profile.md`）就是长期记忆本身。

它不是一个静态文档——每次对话产生有效反馈后，记忆层都会更新。
新开任何一个 Claude Code 窗口，只要读取 `profile.md`，就能恢复所有学到的知识。

---

## 一、会话启动协议（新窗口恢复记忆）

**任何命令**（`/reply` / `/ask` / `/analyze` / ...）被使用时，如果这是一个新的对话窗口，执行以下启动流程：

### 步骤 1：加载档案

读取 `partners/{slug}/profile.md` 中的以下部分：

1. 关系概览（当前阶段、基本信息）
2. 沟通风格和潜台词字典
3. 兴趣度档案和朋友圈/社交动态画像
4. 微信息记忆（MBTI/星座态度/口头禅/妆容穿搭/礼物忌口/心情语气）
5. 表情包/梗偏好记忆
6. 拉扯/节奏策略记忆
7. 海王/海后置信度记录
8. 近期玩家信号记录
9. 追法类型记录
10. 图片结构化留痕摘要（`materials/image_observations.jsonl`，只读取与当前问题相关的图片记录）
11. 分析结论版本（`versions/analysis_vN.md` 的最新版本）
12. **学习记忆层**（核心：从对话中积累的知识）
13. **用户偏好记忆**

### 步骤 2：激活记忆规则

- 置信度 `●●●`（3+次验证）的记忆 → **强制应用**，作为硬性约束
- 置信度 `●●○`（2次验证）的记忆 → **优先参考**，作为强建议
- 置信度 `●○○`（1次观察）的记忆 → **弱参考**，记录但不主导

### 步骤 3：输出恢复摘要（简短，不打扰用户）

```
已加载 {name} 的记忆档案

  当前阶段：{stage_name} · 油腻上限 {N}/5
  已验证策略：{N} 条 ●●●  {N} 条 ●●○
  上次对话：{上次日期 或 "首次对话"}

{如有高置信度记忆，简短说明最关键的一条}
  eg: "记得：{name}在疲惫时最需要「被看穿」类的暖心回复"

现在可以继续：/reply [消息]
```

## 二、记忆结构

记忆写入 `profile.md` 的 `## 学习记忆层` 和 `## 用户偏好记忆` 两个节中。

### 2.1 对象学习记忆层

```markdown
|------|---------|---------|--------|---------|
| ta疲惫/委屈时 | 暖心·被看穿型 | 被心疼 | ●●● | 2026-03 |
| 日常闲聊 | 幽默简短 | 被逗乐 | ●●○ | 2026-03 |
| ta分享成就 | 真诚夸奖·不夸张 | 被崇拜 | ●○○ | 2026-02 |
```

#### 无效模式表

记录什么方式会让这个人冷淡或反应差：

```markdown
### 对 {name} 无效的模式

| 场景 | 无效模式 | 观察到的反应 | 置信度 | 记录时间 |

```

#### 真实模式修正

记录实际观察与初始描述的差异：

```markdown
### 真实模式修正

| 维度 | 初始描述 | 实际观察 | 置信度 | 来源 |
|------|---------|---------|--------|------|
| 活跃时间 | 白天活跃 | 凌晨11-1点主动发消息更多 | ●●○ | 聊天记录分析 |
| 情绪表达 | 不爱表达 | 生气时用省略号，不是沉默 | ●●● | 3次纠正确认 |
```

#### 关系进展日志

```markdown
- [2026-03] 确认关系，阶段 → 4（热恋初期）
```

#### 兴趣度变化记录

记录 ta 是否接球、是否主动、是否值得继续投入。必须拆四维，避免一个总分掩盖行动失败：

```markdown
### 兴趣度变化记录

| 时间 | 事件 | 聊天甜度 | 主动性 | 关系承诺 | 见面/行动兑现 | 证据 | 建议 |
|------|------|----------|--------|----------|----------------|------|------|
| 2026-03 | 用户展示周末生活，ta主动追问 | 6 | 7 | 2 | 1 | ta问"和谁去的"并聊了5轮 | 可继续低油腻轻撩，但别升阶段 |
| 2026-03 | 第3次拒绝见面且不给替代时间 | 4 | 2 | 1 | 0 | "最近都没空" | 反舔狗模式下建议止损 |
```

#### 图片结构化留痕

每张图片的多模态分析记录写入 `materials/image_observations.jsonl`，用于回溯证据。

```json
{"image_id":"img_0001","path":"...","visible_context":"微信聊天截图","initiator":"user","receiver_response":"missed","has_counter_question":false,"has_invite":true,"action_fulfillment":"failed","cooling_signals":["拒约不给替代时间"],"visible_evidence":["对方说最近没空"],"confidence":"medium"}
```

读取规则：

- 当前问题涉及某张或某批图片时，优先读取对应 `image_id` 记录。
- 做总体复盘时，统计主动性、反问、邀约、行动兑现和降温信号的连续模式。
- 不要只凭“看了很多图的感觉”下结论；结论要能回指图片编号。

#### 分析结论版本

完整复盘、阶段重估、兴趣重估或大量导入材料后，写入 `versions/analysis_vN.md`，并在 `profile.md` 的“分析结论版本”表格中追加一行。

版本文件使用 `prompts/analysis_version.md` 模板，必须包含：

- 用户标签和真实阶段校准。
- 四维兴趣分。
- 图片证据索引。
- 反舔狗硬阈值是否触发。
- 下一步验证点。

#### 表情包/梗偏好记忆

记录用户和对象会用、爱用、反感或需要避开的表情包类型和梗。未知梗必须先查公开资料再写入，不能凭感觉编。

```markdown
### 表情包/梗偏好记忆

| 词/图/梗 | 含义 | 风格 | 适合场景 | 来源 | 置信度 | 最近验证 |
|------|------|------|----------|------|--------|----------|
| 奶龙大笑 | 装傻、缓冲尴尬、把暧昧降油 | 抽象发疯 | 熟人轻松互怼 | 公开资料+聊天记录 | ●○○ | 2026-06 |
| 线条小狗 | 委屈、温柔、轻撒娇 | 可爱治愈 | 安慰/晚安/求哄 | 对方常发 | ●●○ | 2026-06 |
```

写入字段：

- 含义：这张图或这个梗在当前聊天里的真实功能。
- 风格：抽象/可爱-线条小狗/可爱-Chiikawa/帅气/长辈图反差/沙雕文字/猫狗治愈等。
- 适合场景：什么时候能用，什么时候别用。
- 来源：聊天记录、朋友圈截图、用户纠正、公开资料。
- 置信度：按 1 次观察、2 次验证、3 次以上验证升级。

### 2.2 用户偏好记忆层

记录用户在实际使用中展示的偏好，而不只是ta自己描述的风格：

```markdown
## 用户偏好记忆

### 策略选择偏好（从实际选择中统计）

| 策略类型 | 选择次数 | 选择率 |
|---------|---------|--------|
| 幽默方案 | 6 | 60% |
| 暖心方案 | 3 | 30% |
| 真诚方案 | 1 | 10% |

→ 偏好：幽默 > 暖心 > 真诚

### 实际修改记录（用户对建议的常见改动）

- 把"宝宝"改成了ta的名字（4/5次）→ 不喜欢用"宝宝"这个词
- 把建议的长回复压缩成一半 → 实际说话比描述的更简短
- 删掉了感叹号 → 语气比描述的更平

### 推断的用户真实风格

基于实际发送（而非自述）得出：
- 实际消息密度：{比自述略短/相符/更长}
- 偏好语气：{冷淡幽默/直接温暖/...}
- 主动程度：{积极主动/被动配合/...}
```

## 三、记忆写入规则

### 3.1 触发时机

以下事件触发记忆写入：

| 事件 | 写入内容 | 置信度变化 |
|------|---------|-----------|
| 用户说"效果很好" | 当次策略 → 有效表 | +1次 |
| 用户上传聊天记录，效果判断为"非常好/好" | 当次策略 → 有效表 | +1次 |
| 用户说"ta没回"/"效果不好" | 当次策略 → 无效表 | +1次 |
| 用户上传聊天记录，效果判断为"较差/没效果" | 当次策略 → 无效表 | +1次 |
| 用户纠正"ta不是这样的" | 对应字段 → 真实模式修正表 | 更新 |
| 用户说"我们确认关系了" | 关系进展日志 | 新增条目 |
| 用户实际选择某个方案 | 用户选择偏好 | +1次计数 |
| 对方接住轻撩/主动追问用户展示 | 兴趣度变化记录 | +1 或 +2 |
| 对方连续礼貌敷衍/拒约不给替代时间 | 兴趣度变化记录 | -1 或 -2 |
| 输出四维兴趣判断 | 兴趣度变化记录 + meta.json interest dimensions | 更新 |
| 对大量材料做阶段/兴趣重估 | versions/analysis_vN.md + profile.md 分析结论版本 | 新增 |
| 使用策略方框后用户反馈结果 | 拉扯/节奏策略记录 | 新增或更新 |
| 判断出海王/海后置信度 | 海王/海后置信度记录 | 新增或更新 |
| 判断出追法类型或拉扯强度 | 追法类型记录 | 新增或更新 |
| 命中近期玩家信号 | 近期玩家信号记录 | 新增或更新 |
| 用户上传朋友圈截图 | 朋友圈/社交动态画像记录 | 新增或更新 |
| 用户上传或导入图片 | materials/image_observations.jsonl | 每图新增或更新 |
| 从图片/聊天提取到 MBTI、星座、口头禅、妆容、礼物、忌口 | 微信息记录 | 新增或更新 |
| 用户/对象使用或提到表情包、热梗、IP、黑话 | 表情包/梗偏好记忆 | 先查证，再新增或更新 |
| 模型遇到不知道的梗并完成上网查证 | 表情包/梗偏好记忆 | 记录含义、风格、来源和日期 |
| 发起或接受邀约 | 约会偏好/关系进展日志 | 新增或更新 |
| 用户开启/关闭反舔狗模式 | meta.json + profile.md 概览 | 更新 |
| 用户提到纪念日、生日、第一次见面等重要日期 | 概览的「重要日子」 | 新增或更新，临近时主动提醒 |

### 3.2 置信度升级规则

```
初次观察 → ●○○（弱参考）
同一模式第2次确认 → ●●○（强参考）
同一模式第3次以上确认 → ●●●（强制应用）
```

同一模式 = 相似场景 + 相似策略 + 相似结果

### 3.3 冲突处理

当新观察与已有记忆冲突时：

- **高置信度记忆被推翻**（●●● 被否定）：降级为 ●○○，加注 `[冲突，待观察]`
- **低置信度记忆被推翻**（●○○）：直接移到无效表
- **可能是情境差异**：保留双条记录，标注适用条件

例：
```
原记忆：日常闲聊用幽默效果好 ●●●
新观察：ta情绪低时用幽默，ta没回
处理：细化为"日常闲聊·情绪正常时 幽默有效 ●●●"
      新增："ta情绪低时 幽默无效 ●○○"
```

### 3.4 写入格式

每次写入都追加到对应表格，不删除旧记录（只更新置信度和最近验证日期）。

日期格式：`YYYY-MM`（月份粒度，不需要精确到天）。

## 四、记忆读取规则

生成每次建议时，按以下顺序应用记忆：

```
优先级 1：●●● 有效策略 → 同场景下优先推荐这个策略
优先级 2：●●● 无效模式 → 同场景下绝对不推荐这类建议
优先级 3：●●○ 有效策略 → 优先参考，可推荐
优先级 4：●●○ 无效模式 → 尽量避免
优先级 5：●○○ 记录 → 弱参考，不主导
```

**用户偏好记忆的应用**：
- 如果用户明显偏好某类策略（选择率 >50%），该类型放在第一位
- 如果用户有明显的修改习惯（如总删感叹号），生成时直接去掉

**兴趣度记忆的应用**：
- 先看四维分数，不只看总体分。
- 聊天甜度高、主动性低：减少追问，改展示自己或轻推回球。
- 主动性高、行动兑现低：可以聊，但下一步必须做行动测试。
- 关系承诺低于 3：不要输出热恋、双向奔赴或确定关系。
- 见面/行动兑现低于 3：优先邀约测试、降频观察或止损，不继续堆情绪价值。
- 总体分低于 4：优先展示自己和降温，不继续查户口式追问。
- 总体分 5-6：用低油腻轻撩测试接球。
- 总体分高于 7 且关系承诺、行动兑现不低：可以推进见面或更明确的暧昧。
- 反舔狗模式开启且总体分 0-2、关系承诺 0-2、行动兑现 0-2，或出现结论性止步证据：直接建议止损、体面收尾或换下一个。

**拉扯/节奏策略记忆的应用**：
- 生成回复时优先参考对这个对象有效的回复间隔。
- 如果“短暂停顿/降频观察”多次有效，策略方框里优先推荐。
- 如果用户停太久导致冷场，下次缩短间隔。
- 如果用户秒回后对方明显降温，下次延迟回复或本轮收尾。

**海王/海后置信度记录的应用**：
- 分数升高：减少主动投入，优先邀约测试、轻推回球、降频观察。
- 分数降低：可以按四维兴趣分继续推进。
- 分数 61+ 且反舔狗模式开启：策略框默认给“暂停主动/等对方行动”。

**追法类型记录的应用**：
- 直球/慢热/事业目标型：回复更清楚具体，少玩猜谜。
- 审美/仪式感/情绪细腻型：夸细节、接情绪、设计体验。
- 社交活跃/幽默游戏型：反差、轻互怼、留钩子。
- 高选择权/玩家型：少捧，多筛选；优先轻推回球和行动测试。
- 已验证有效的拉扯强度优先使用；无效强度下次降一级或换打法。

**近期玩家信号记录的应用**：
- 模板化高情绪价值：优先做专属细节测试，少被漂亮话带走。
- 平台人设型：夸具体审美后做低成本邀约测试。
- 多线排班型：换时段测试，减少主动投喂。
- 评论区暧昧型：做私聊兑现测试，不把公开热闹当高兴趣。
- 节日借势型：做节日兑现测试，看是否给具体安排。
- 朋友圈定向投喂：轻推成私聊行动，看是否接。

**微信息记忆的应用**：
- 邀约前检查忌口、作息、地点偏好、仪式感偏好
- 送花/礼物前检查对方是否喜欢仪式感、花材颜色、过敏和雷区
- 聊天时复用对方口头禅和梗，但不要硬学到油
- MBTI/星座只当聊天梗和偏好，不当人格判断依据
- 心情相关口头禅用于识别“开心/累/emo/生气”状态

**表情包/梗偏好记忆的应用**：
- 可复制回复默认只写文字。
- 需要补语气时，只在“表情包建议（不要复制给 ta）”里给类型建议。
- 对方喜欢抽象梗时，可以用抽象发疯、奶龙类、沙雕文字降油。
- 对方喜欢治愈可爱时，可以用线条小狗、Chiikawa、猫狗类表达温柔。
- 不确定某个梗或 IP 时，先上网查；平台不能查时，宁可不用。
- 对方明显低兴趣时，不用表情包继续求回应。

### 记忆老化

超过 **3个月** 未被验证的 `●○○` 记忆，在展示时标注 `[待验证]`，不再主动应用。
`●●○` 和 `●●●` 记忆不老化，除非用户明确否定。

## 六、跨窗口连续性保证

**这是系统的核心保证**：

只要 `partners/{slug}/profile.md` 文件存在，**任何新的 Claude Code 窗口都能恢复完整的上下文和学习到的知识**。

用户不需要重新解释对象的特点。
用户不需要反复解释"上次已经知道她不喜欢长回复"。
不需要导入任何对话历史。

文件就是记忆。记忆随对话成长。

---

## 七、每个对象各存各的

每个对象一个 `partners/{slug}/` 文件夹，记忆互不干扰；新建对象就是开一个空文件夹，不会读到别人的记忆。删掉某个文件夹，就等于清掉那个对象的全部记忆。
