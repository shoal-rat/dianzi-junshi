# 长期记忆引擎

## 设计原则

档案文件（`partners/{slug}/profile.md`）就是长期记忆本身。

它不是一个静态文档——每次对话产生有效反馈后，记忆层都会更新。
新开任何一个 Claude Code 窗口，只要读取 `profile.md`，就能恢复所有学到的知识。

---

## 一、会话启动协议（新窗口恢复记忆）

**任何命令**（`/reply` / `/ask` / `/analyze` / ...）被使用时，如果这是一个新的对话窗口，执行以下启动流程：

### 步骤 1：加载档案

读取 `partners/{slug}/profile.md` 中的以下部分：

1. 关系概览（当前阶段、基本信息）
2. 沟通风格和潜台词字典
3. **学习记忆层**（核心：从对话中积累的知识）
4. **用户偏好记忆**

### 步骤 2：激活记忆规则

- 置信度 `●●●`（3+次验证）的记忆 → **强制应用**，作为硬性约束
- 置信度 `●●○`（2次验证）的记忆 → **优先参考**，作为强建议
- 置信度 `●○○`（1次观察）的记忆 → **弱参考**，记录但不主导

### 步骤 3：输出恢复摘要（简短，不打扰用户）

```
🧠 已加载 {name} 的记忆档案

  当前阶段：{stage_name} · 油腻上限 {N}/5
  已验证策略：{N} 条 ●●●  {N} 条 ●●○
  上次对话：{上次日期 或 "首次对话"}

{如有高置信度记忆，简短说明最关键的一条}
  eg: "记得：{name}在疲惫时最需要「被看穿」类的暖心回复"

现在可以继续：/reply [消息]
```

## 二、记忆结构

记忆写入 `profile.md` 的 `## 学习记忆层` 和 `## 用户偏好记忆` 两个节中。

### 2.1 对象学习记忆层

#### 有效策略表

记录什么策略在什么场景下对这个人有效：

```markdown
### 对 {name} 有效的策略

| 场景 | 有效策略 | 情感价值 | 置信度 | 最近验证 |
|------|---------|---------|--------|---------|
| ta疲惫/委屈时 | 暖心·被看穿型 | 被心疼 | ●●● | 2026-03 |
| 日常闲聊 | 幽默简短 | 被逗乐 | ●●○ | 2026-03 |
| ta分享成就 | 真诚夸奖·不夸张 | 被崇拜 | ●○○ | 2026-02 |
```

#### 无效模式表

记录什么方式会让这个人冷淡或反应差：

```markdown
### 对 {name} 无效的模式

| 场景 | 无效模式 | 观察到的反应 | 置信度 | 记录时间 |
|------|---------|------------|--------|---------|
| 任何场景 | 长篇解释 | 回复变短，话题转移 | ●●○ | 2026-02 |
| ta情绪低时 | 讲道理/给建议 | "嗯" 结束对话 | ●●● | 2026-03 |
```

#### 真实模式修正

记录实际观察与初始描述的差异：

```markdown
### 真实模式修正

| 维度 | 初始描述 | 实际观察 | 置信度 | 来源 |
|------|---------|---------|--------|------|
| 活跃时间 | 白天活跃 | 凌晨11-1点主动发消息更多 | ●●○ | 聊天记录分析 |
| 情绪表达 | 不爱表达 | 生气时用省略号，不是沉默 | ●●● | 3次纠正确认 |
```

#### 关系进展日志

记录关系中的关键节点：

```markdown
### 关系进展日志

- [2026-01] 初识，通过xxx认识，阶段 0
- [2026-02] 开始频繁联系，阶段 → 1（暧昧期）
- [2026-02] 首次线下见面
- [2026-03] 确认关系，阶段 → 4（热恋初期）
```

### 2.2 用户偏好记忆层

记录用户在实际使用中展示的偏好，而不只是ta自己描述的风格：

```markdown
## 用户偏好记忆

### 策略选择偏好（从实际选择中统计）

| 策略类型 | 选择次数 | 选择率 |
|---------|---------|--------|
| 幽默方案 | 6 | 60% |
| 暖心方案 | 3 | 30% |
| 真诚方案 | 1 | 10% |

→ 偏好：幽默 > 暖心 > 真诚

### 实际修改记录（用户对建议的常见改动）

- 把"宝宝"改成了ta的名字（4/5次）→ 不喜欢用"宝宝"这个词
- 把建议的长回复压缩成一半 → 实际说话比描述的更简短
- 删掉了感叹号 → 语气比描述的更平

### 推断的用户真实风格

基于实际发送（而非自述）得出：
- 实际消息密度：{比自述略短/相符/更长}
- 偏好语气：{冷淡幽默/直接温暖/...}
- 主动程度：{积极主动/被动配合/...}
```

## 三、记忆写入规则

### 3.1 触发时机

以下事件触发记忆写入：

| 事件 | 写入内容 | 置信度变化 |
|------|---------|-----------|
| 用户说"效果很好" | 当次策略 → 有效表 | +1次 |
| 用户上传聊天记录，效果判断为"非常好/好" | 当次策略 → 有效表 | +1次 |
| 用户说"ta没回"/"效果不好" | 当次策略 → 无效表 | +1次 |
| 用户上传聊天记录，效果判断为"较差/没效果" | 当次策略 → 无效表 | +1次 |
| 用户纠正"ta不是这样的" | 对应字段 → 真实模式修正表 | 更新 |
| 用户说"我们确认关系了" | 关系进展日志 | 新增条目 |
| 用户实际选择某个方案 | 用户选择偏好 | +1次计数 |

### 3.2 置信度升级规则

```
初次观察 → ●○○（弱参考）
同一模式第2次确认 → ●●○（强参考）
同一模式第3次以上确认 → ●●●（强制应用）
```

同一模式 = 相似场景 + 相似策略 + 相似结果

### 3.3 冲突处理

当新观察与已有记忆冲突时：

- **高置信度记忆被推翻**（●●● 被否定）：降级为 ●○○，加注 `[冲突，待观察]`
- **低置信度记忆被推翻**（●○○）：直接移到无效表
- **可能是情境差异**：保留双条记录，标注适用条件

例：
```
原记忆：日常闲聊用幽默效果好 ●●●
新观察：ta情绪低时用幽默，ta没回
处理：细化为"日常闲聊·情绪正常时 幽默有效 ●●●"
      新增："ta情绪低时 幽默无效 ●○○"
```

### 3.4 写入格式

每次写入都追加到对应表格，不删除旧记录（只更新置信度和最近验证日期）。

日期格式：`YYYY-MM`（月份粒度，不需要精确到天）。

---

## 四、记忆读取规则

生成每次建议时，按以下顺序应用记忆：

```
优先级 1：●●● 有效策略 → 同场景下优先推荐这个策略
优先级 2：●●● 无效模式 → 同场景下绝对不生成这类建议
优先级 3：●●○ 有效策略 → 优先参考，可推荐
优先级 4：●●○ 无效模式 → 尽量避免
优先级 5：●○○ 记录 → 弱参考，不主导
```

**用户偏好记忆的应用**：
- 如果用户明显偏好某类策略（选择率 >50%），该类型放在第一位
- 如果用户有明显的修改习惯（如总删感叹号），生成时直接去掉

## 五、记忆维护

### /memory 命令

用户可以用 `/memory` 查看当前档案的记忆状态：

```
📚 {name} 的记忆档案

有效策略（{N}条）：
  ●●● {N}条  ●●○ {N}条  ●○○ {N}条

无效模式（{N}条）：
  ...

真实模式修正：{N}条

用户偏好：
  最常选择：{策略类型}
  实际风格修正：{N}条

关系进展：{最新节点}

上次写入：{日期}

/memory delete [条目编号]  — 删除某条错误记忆
/memory export            — 导出完整记忆摘要
```

### 记忆老化

超过 **3个月** 未被验证的 `●○○` 记忆，在展示时标注 `[待验证]`，不再主动应用。
`●●○` 和 `●●●` 记忆不老化，除非用户明确否定。

---

## 六、跨窗口连续性保证

**这是系统的核心保证**：

只要 `partners/{slug}/profile.md` 文件存在，**任何新的 Claude Code 窗口都能恢复完整的上下文和学习到的知识**。

用户不需要重新解释对象的特点。
用户不需要告诉 AI"上次你知道她不喜欢长回复"。
不需要导入任何对话历史。

文件就是记忆。记忆随对话成长。

---

## 七、隐私说明

所有记忆存储在本地 `partners/{slug}/profile.md` 文件中。
此文件不会被 git 追踪（`.gitignore` 已排除 `partners/` 目录）。
用户删除文件即等同于清除所有记忆。
