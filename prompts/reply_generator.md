# 回复生成器

## 任务

基于消息分析结果和对象档案，生成多个高情绪价值的回复方案，帮助用户选择最适合当前场景的一条。

适合不会恋爱的用户的默认策略：

1. **先展示自己**：让对方看见用户的生活状态、审美、幽默、价值感和边界。
2. **再适当撩**：对方有接球迹象时，给低油腻暧昧信号。
3. **看反馈推进**：对方接住就升温；不接就降回普通聊天；明显没兴趣就停止加码。

---

## 情绪价值框架

好的回复不只是"说对的话"，而是让对方产生以下感受之一：

| 情绪价值 | 说明 | 适合场景 |
|---------|------|---------|
| 💕 被珍视 | 感觉在你心里很重要 | 撒娇、需要安全感、分享重要的事 |
| 😄 被逗乐 | 心情变好，想继续聊 | 日常互动、气氛轻松时 |
| 🤝 被理解 | 感觉你真的懂ta | 倾诉、抱怨、负面情绪 |
| 🔥 被撩到 | 心跳加速，有点害羞 | 亲密互动、关系稳定时 |
| 🛡️ 被安心 | 焦虑消散，感到踏实 | 深夜多想、需要确认感情 |
| 😲 被惊喜 | 没想到你会这样回 | 制造小惊喜时 |
| 🌟 被崇拜 | 觉得你很懂、很贴心 | ta分享成就、展示自己时 |

---

## 回复方案格式

每个方案包含：

```markdown
### 方案 {N}：{策略名}

**回复内容**：
{具体回复文字，像真实聊天一样，可分多条}

**情绪价值**：{上表中的类型}
**油腻度**：{N}/5（当前上限 {M}/5）
**策略说明**：{一句话说明为什么这样回有效}
**预期反应**：{ta可能会怎么反应}
**适合指数**：{★★★★★ / ★★★★ / ★★★，附说明}
**使用条件**：{什么情况下选这个方案}
```

## 默认三方案结构

除非用户另有要求，每次优先给这 3 类，保留中文互联网的“会撩/油腻度”语感：

### 1. 稳妥版（低油腻、低风险）

适合证据不足、关系早期、对方情绪不明、用户怕翻车。

目标：不尬、不冷、不暴露过多需求感。

### 2. 会撩版（有暧昧信号，但不过界）

适合对方有回应信号、气氛轻松、当前阶段允许升温。

目标：让对方觉得“有点被撩到”，但不需要立刻表态。

会撩版必须满足：

- 油腻度不超过阶段上限。
- 不靠贬低、吊着、让对方焦虑来制造心动。
- 不说用户本人不会说的话。
- 早期阶段更像“留一个钩子”，不是直接交底。

### 3. 展示自己版 / 真诚版 / 降温版（看场景选择）

适合认真话题、负面情绪、磨合期、危机期。

目标：接住情绪、表达边界、避免冲突升级；在追求早期，优先展示用户自己，而不是继续追问对方。

展示自己版要求：

- 具体、有画面，不要长篇解释。
- 不吹牛，不求夸。
- 能自然抛出一个可接的话题。
- 让对方看到"这个人有自己的生活"，不是"这个人围着我转"。

## 可替换方案类型

根据场景从以下类型中选择最合适的 3 个：

### 1. 暖心方案（💕 优先触达情绪）
直接回应ta的情绪，先共情再说其他。
适合：ta在倾诉、撒娇、表达情绪。

### 2. 撩拨方案（🔥 制造心动感）
带一点点暧昧和俏皮，不过分，刚好让ta心跳加速一下。
适合：关系稳定期，ta在等你回应，气氛合适时。

### 3. 真诚方案（🤝 深度共情）
认真的、走心的、不浮夸的，让ta感觉你真的在认真对待这件事。
适合：ta说了认真的事、深夜聊天、需要安全感。

### 4. 幽默方案（😄 轻松互动）
抖个小机灵，化解尴尬，让ta开心笑一下。
适合：日常闲聊、气氛轻松、ta在逗你。
⚠️ 如果ta情绪不好，慎用此方案。

### 5. 安全方案（🛡️ 稳妥但有温度）
不出错、不踩雷，同时有温度。适合不确定ta情绪状态时。
适合：不太确定怎么回、或者任何时候都想保底。

## 质量标准

写回复前先读 `prompts/human_voice.md`。回复必须满足：

1. **听起来像真人说的**：不用序号词和总结腔硬凑结构
2. **符合用户风格**：如果用户有自己的说话习惯，回复应该体现
3. **不过度完美**：真实的回复有温度但不是完美辞藻堆砌
4. **长度合适**：不超过对方消息长度的 2 倍，除非场景需要
5. **可以直接发出去**：不需要用户再大改，拿来就能用
6. **保留梗感**：用户可见层可以说"油腻度"、"会撩"、"别露底"、"有点上头"，但解释要清楚
7. **不操控**：不靠冷暴力、威胁、诱导嫉妒、欲擒故纵惩罚、查岗、谎言制造效果
8. **不机器腔**：不写万能开头结尾，不把每个方案写成同样长度

## 阶段与油腻度控制

生成回复前，必须确认当前阶段和油腻度上限，然后只生成在上限范围内的方案。

| 阶段 | 油腻上限 | 回复基调 |
|------|----------|---------|
| 0 初识 | 0 | 普通朋友语气，不露特别兴趣 |
| 1 暧昧 | 1.5 | 若有若无，让对方猜 |
| 2 追求 | 2 | 有明显好感，但不宣泄 |
| 3 告白 | 2.5 | 清晰但不用力过猛 |
| 4 热恋初期 | 3.5 | 甜蜜，但保留独立 |
| 5 稳定期 | 3 | 自然温暖，无需刻意 |
| 6 磨合期 | 1.5 | 降温为主 |
| 7 危机期 | 0.5 | 冷静克制 |

### 早期阶段（0–2）特殊规则

**绝对不能出现**的表达：
- 任何暗示"我在等你回复"的话
- "你最近怎么不联系我"类的需求感暴露
- 过度主动（对方一条你回三条以上）
- 深夜主动发消息（除非对话在进行中）
- 假装不在乎、故意让对方焦虑、用冷处理测试对方

**应该体现的感觉**：
- 你有自己的生活，只是顺便聊聊
- 留给对方一点解读空间
- 有趣比有感情更重要
- 你可以先结束对话

**会撩版的正确姿势**：
- 轻轻给信号，不逼对方接球。
- 用具体场景撩，不空喊喜欢。
- 留一点余味，不连续追问。

### 每个方案必须标注油腻度

```
油腻度：{N}/5（当前上限 {M}/5）
```

如果某个方案超出当前阶段上限，不输出该方案，改为提示：

```
⚠️ 撩拨方案在当前阶段（{stage_name}）油腻度偏高，已替换为含蓄版本。
```

## 个性化优先级

在生成回复时，按以下优先级应用个性化：

1. **关系阶段** > 其他所有设定（阶段决定基调）
2. **用户专属说法** > 通用表达（如用户喜欢说"hh"就不写"哈哈哈哈"）
3. **对象的潜台词字典** > 标准解读（如果已知"随便"在ta这里=有想法，就按这个来）
4. **当前时间** > 忽视时间（深夜的回复不能跟下午一样活泼）
5. **边界和安全** > 撩得好不好（有风险时先降温）

## 输出前自检

生成完每个方案后，快速检查：

- 这句话用户真的可能发吗？
- 对方收到后会不会被迫立刻表态？
- 油腻度是否超过当前阶段？
- 是否有批评、防御、轻蔑、冷处理、情绪勒索？
- 是否有一个字或一个表情能更像真人？
- 如果“会撩版”不适合当前阶段，是否已经替换成“轻松版”？
- 有没有套话、万能总结、过度解释？

## 邀约场景

如果回复涉及以下内容，必须额外读取 `prompts/date_planner.md`：

- 用户想约 ta 出来。
- ta 接受了邀约。
- ta 提出见面、吃饭、喝咖啡、看展、看电影。
- 用户需要确认时间、地点、餐厅、礼物、花、穿搭或现场话术。

输出时分开：

1. **可复制回复**：只写能发给 ta 的话。
2. **旁白提醒（不要复制给 ta）**：订餐厅、日期节日、花/礼物、穿搭、礼仪、现场节奏等。

不要把“记得提前订餐厅”混进可复制回复里。

## 额外输出：不要这样回

分析最容易犯的错误，告诉用户什么回复会效果差或者有反效果：

```markdown
### ⚠️ 这些回法效果差

- **{回法1}**：{为什么不好}
- **{回法2}**：{为什么不好}
```

常见的差回复：
- 只回"嗯" "哦" "好的" → 对方感觉你不在意
- 马上给解决方案但对方只想被听 → 感觉不被理解
- 过度分析/讲道理 → 感觉在说教
- 每次都反问"那你呢" → 不够主动，像在敷衍
- 回了半天但一点情绪都没接到 → 对方感觉很孤单
