# 自动效果分析

## 任务

用户发完消息之后，可以上传后续的聊天记录，让系统自动分析这次回复是否有效，记录下来用于优化未来的建议。

---

## 触发方式

```
/upload-followup
/反馈记录
```

或者用户说：
- "我上传一下后续聊天记录"
- "发完之后的对话是这样的"
- "ta的反应是..."（附上截图或文字）

---

## 分析流程

### 步骤 1：提取上次建议的内容

回溯当前对话中最近一次 `/reply` 生成的方案，识别：
- 用户选择了哪个方案（或修改后的版本）
- 该方案的核心策略和情感价值类型

### 步骤 2：解析后续聊天记录

从用户上传的内容中提取：
- 用户实际发送的消息（可能与方案有所不同）
- 对方的回应内容
- 后续对话是否持续（互动轮次）
- 对方的情绪变化迹象

支持输入格式：
- 直接粘贴聊天内容
- 上传截图（从截图文字中提取）
- WeChatMsg 导出格式

### 步骤 3：效果判断

根据对方回应分析效果：

| 效果等级 | 判断标准 |
|---------|---------|
| 非常好 | 对方明显情绪变好，话变多，主动延续对话 |
| 好 | 对方正常回应，对话继续，氛围良好 |
| 中性 | 对方回了，但对话没有特别推进 |
| 较差 | 对方回应简短、冷淡，或改变了话题 |
| 没效果 | 对方没回，或明显情绪变差 |

### 步骤 4：策略归因

判断这次效果是否与所用策略相关：
- **策略加分点**：哪个词/句式引发了积极反应
- **策略风险点**：哪个表达可能导致了冷淡回应
- **情境因素**：效果与策略无关（如对方当时很忙）

### 步骤 5：记录到档案

将以下信息记录到 session log：

```json
{
  "session_id": "timestamp",
  "message_analyzed": "原始消息内容",
  "option_used": "方案N / 用户自改版本",
  "strategy": "策略类型",
  "emotional_value_target": "目标情感价值",
  "actual_reply": "用户实际发送的话",
  "partner_response": "对方回应摘要",
  "follow_up_turns": 对话持续轮次,
  "outcome": "非常好/好/中性/较差/没效果",
  "attribution": "策略有效/策略有风险/情境因素",
  "notes": "分析备注"
}
```

## 累积分析

当某个对象累计有 5 条以上有效果记录时，系统可以生成策略总结：

```
/stats
```

输出该对象的策略效果统计：
- 哪类策略对ta最有效
- ta最常触发的正面情绪类型
- 哪类表达会让ta冷淡
- 发消息的最佳时间段

---

## 无需上传的轻量反馈

如果用户只是口头说结果，也记录：

- "效果很好" → 记录当前方案为 outcome: 好
- "ta没回" → 记录为 outcome: 没效果
- "ta回了但感觉冷冷的" → 记录为 outcome: 较差
- "一直在聊" → 记录为 outcome: 非常好

这类轻量反馈会被记录，但置信度标注为"主观报告"，权重低于从聊天记录解析的数据。
