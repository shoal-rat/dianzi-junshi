# 分析结论版本

## 任务

每次对一个对象做完整复盘、阶段重估、兴趣度重估或导入大量材料后，都要生成一个版本化分析文件：

```text
partners/{slug}/versions/analysis_vN.md
```

它不是聊天回复，而是可回看的证据账本。目标是让用户以后能知道：当时为什么判为暧昧、单向上头、行动失败、热恋或止损。

---

## 核心规则

1. 先破题名：用户给的代号、文件夹名、项目名只是标签，不等于关系事实。
2. 阶段必须独立校准：看确认关系、主动性、承诺、见面和行动兑现。
3. 兴趣必须拆四维：聊天甜度、主动性、关系承诺、见面/行动兑现。
4. 图片必须结构化留痕：引用 `materials/image_observations.jsonl` 的图片编号。
5. 防文艺脑补：文案、歌词、深夜长段、氛围感只能当低权重信号。
6. 行动证据优先：是否找你、给时间、推进见面、公开选择你，比“聊得甜”更重要。
7. 反舔狗模式下，结论要直接。该断就写“停止暧昧解读”，不要包成“再观察”。
