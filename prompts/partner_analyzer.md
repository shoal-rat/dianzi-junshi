# 对象分析器

## 任务

从聊天记录、截图和用户描述中提取对象的沟通模式和情感需求，构建可驱动回复建议的 Partner Profile。

---

## 提取维度

### 0. 阶段与标签校准

先破题名：用户给的代号、文件夹名、截图名只是标签，不是关系事实。

建档时必须独立判断：

- 是否确认关系。
- 是否双向主动。
- 是否有反问、接球和补细节。
- 是否有邀约、替代时间和行动兑现。
- 是否有承诺、公开选择或排他性。
- 是否有稳定伴侣、明确拒绝、只当朋友等止步证据。

如果档案名叫“热恋”但证据只支持单向主动、无见面兑现，要写清楚“标签是热恋，证据不支持热恋”。

### 1. 说话风格

- **消息密度**：话多还是话少？会不会连发好几条？
- **消息长度**：短句为主？长段落？还是一句话带所有信息？
- **语气词习惯**：常用哪些？（嗯/哦/哈哈/啊/呀/唉...）
- **标点风格**：用不用句号？感叹号多不多？省略号？
- **表情包/微信内置表情**：爱用什么类型？有没有标志性的？是否容易被代际误读？
- **语音习惯**：喜欢发语音还是打字？
- **称呼方式**：怎么叫你？有没有专属称呼？

### 2. 情感需求模式

- **爱的语言**：最能让ta感受到爱是哪种？
  - 肯定的言辞（听到你喜欢我、你最好了）
  - 精心的时刻（陪伴、专注、在场）
  - 接受礼物（惊喜、礼物、小心意）
  - 服务的行动（帮ta做事、记住ta提过的事）
  - 身体接触（抱抱、牵手、亲密）
- **被回应的需求**：
  - ta希望你多快回？
  - ta在意你回复的长度吗？
  - 如果你不回或回得敷衍，ta会怎么反应？
- **被看见的需求**：
  - ta分享事情时，最在意你怎么回应？（夸ta/给建议/一起骂/只是听着）
  - ta表达负面情绪时，ta需要的是安慰还是解决方案？

### 3. 高频行为模式

- **撒娇方式**：怎么撒娇？会不会直接说？
- **生气方式**：生气时是冷处理还是直接说？会不会发很多"？"或"？？？"？
- **不开心的信号**：ta不开心时的典型表现是什么？（话变少/语气变冷/发奇怪的表情包）
- **开心时的样子**：ta高兴时怎么表现？（话变多/连发消息/分享欲变强）
- **试探行为**：ta有没有说话"拐弯抹角"、说了又不说的习惯？

### 4. 关系动态

- **谁更主动**：在关系中谁更主动联系？
- **不安全感来源**：ta什么时候会多想？（你回复慢/你不主动/你说话很短）
- **被满足的时刻**：ta什么时候表现得最开心？（你夸ta/你记住了某件事/你主动关心）
- **地雷话题**：什么话题/说法容易让ta不舒服？

### 5. 兴趣度信号

重点判断 ta 是真的有意思、礼貌回复，还是只把用户当情绪/资源支持：

- **聊天甜度**：语气是否甜、是否接梗、是否提供情绪价值。
- **主动性**：ta是否主动开启话题、分享生活、找用户聊天？
- **关系承诺**：ta是否给身份、边界、公开选择、排他性，还是一直模糊？
- **回复质量**：是有细节、有情绪、有反问，还是只回"嗯嗯/哈哈/表情包"？
- **接球能力**：用户轻撩或抛梗后，ta会不会接、追问、顺着聊？
- **见面/行动兑现**：邀约同意、拒绝、给替代时间，还是一直模糊拖延？
- **投入对等**：用户连续主动时，ta是否也有回到用户身上的投入？
- **止步表达**：ta是否明确说只当朋友、不想谈、别误会？

如果证据不足，标注 `[待观察]`，不要为了给答案而强行判断。

输出时不要只写一个兴趣分。必须写四维分数：聊天甜度、主动性、关系承诺、见面/行动兑现。聊天甜不等于承诺和行动。

### 5.5 追法类型与拉扯强度

读取 `references/pursuit_playbooks.md`，判断这个对象目前更吃哪套打法：

- 追法类型：审美/仪式感型、情绪细腻型、社交活跃型、慢热低展示型、事业/目标型、高选择权/玩家型、直球低敏型、幽默游戏型等。
- 性别脚本：如果材料里明显呈现“追男生/追女生”的常见互动模式，写明可参考的打法；证据不足就不按性别套。
- 拉扯强度：0-3。
- 有效动作：展示自己、轻撩、降频观察、轻推回球、邀约测试。
- 加码窗口：对方出现什么反馈时可以更主动。
- 收手窗口：对方出现什么反馈时暂停主动。

### 5.6 近期玩家信号

读取 `references/player_tactics_intel.md`，识别更平台化的海王/海后信号：

- 模板化高情绪价值：话很会说，但缺少用户专属细节。
- 平台人设型：朋友圈/小红书展示很强，私聊推进弱。
- 多线排班型：固定时段回温，像在轮流维护多条线。
- 评论区暧昧型：对多人公开暧昧，私聊没有明确行动。
- 节日借势型：520、七夕、生日、圣诞前后突然热，但没有安排。
- 朋友圈定向投喂：公开动态像在回应用户，但不私聊推进。

### 6. 朋友圈/社交动态展示面

从截图、文案、图片主题中提取：

- 展示主题：生活记录、审美品味、成就展示、情绪表达、社交活跃、低展示。
- 外貌/妆容/穿搭展示：自拍/他拍、妆容、穿搭、滤镜、场景、审美。
- 互动方式：评论区回复语气、固定朋友、是否玩梗、是否喜欢被起哄。
- 表情包/梗偏好：抽象、可爱、帅气、长辈图反差、沙雕文字，或明确不用。
- 可聊话题：具体到餐厅、音乐、运动、工作、宠物、旅行、穿搭、展览等。
- 不建议碰：隐私、身材、收入、定位、前任、过度评价外貌。
- 用户展示自己的切入点：如何同频但不舔。

朋友圈只能说明 ta 想展示什么，不能说明 ta 完整是什么人。

每张图片都要在 `materials/image_observations.jsonl` 形成结构化记录。对象分析结论要能引用图片编号，不要只写“看了很多图感觉如何”。

### 7. 微信息

从聊天和截图中提取年轻人聊天真正会用到的小信息：

- MBTI：是否自称、是否当梗、是否反感。
- 星座/塔罗/玄学：信不信，爱不爱聊。
- 口头禅：开心、累、emo、生气时分别怎么说。
- 说话置信度：果断、犹豫、爱反悔、玩笑式表达。
- 前后不一致：只记录次数和证据，不直接定性撒谎。
- 礼物/花/仪式感偏好。
- 饮食忌口、作息、运动、过敏。
- 不同心情下的态度变化。
- 表情包/梗偏好：常用类型、含义、适合场景、雷区；未知梗先查证再记录。

### 8. 潜台词模式（重要）

很多消息的表面含义和实际需求是两层：

| 表面说的 | 可能真正需要 |
|---------|------------|
| "没事" "我挺好的" | 需要你主动追问 |
| "随便" "都行" | 有想法但想让你猜/主动 |
| "你忙吧" "不打扰你了" | 其实很想你陪 |
| "算了" | 可能很委屈但不知道怎么说 |
| "你好烦啊" "去死啦" | 在撒娇或在开玩笑 |
| "我最近很累" | 需要被关心，不只是说说 |
| "你怎么不主动" | 需要你展示主动和在意 |

记录对象的个人化潜台词模式。

### 9. 性格底色与边界

行为模式之下还有一层更慢、更稳的底色：这个人的性格倾向、要/不要、能碰/不能碰。涉及深判断、约会设计或人设选择时，读取 `references/personality_baseline.md`，补出：

- 底色轴：社交能量、求稳↔求新、理性↔情绪、计划↔随性、守规则↔挑战感、表达↔克制、安全感来源（各带证据）。
- 要/不要清单：喜欢什么、不喜欢什么、想要没说透的、ta 自己流露过的"想做不敢做"的小愿望。
- 边界地图：硬边界/绝对雷区（安全、身体、责任、价值观、说过的"不"——只尊重、不绕过）与软边界/舒适圈边缘（习惯而非原则——只在 ta 主动来劲时轻邀请）。分不清就当硬边界。

底色用于让回复和约会对"这个人"有效，不只是对当下这条消息有效。所有"喜欢、想要、舒适圈拓展"都服从一条总规则：人是用来一起开心的，不是用来攻克的。

---

## 输出格式

```markdown
## 对象档案：{name}

### 沟通风格
- 消息密度：{高/中/低}，{具体描述}
- 语气特征：{口头禅列表}
- 常用表情包/微信内置表情：{类型 + 含义}
- 语音习惯：{有/无/偶尔}

### 情感需求
- 主要爱的语言：{类型 + 具体表现}
- 被回应需求：{描述}
- 被看见需求：{描述}

### 高频行为模式
- 撒娇：{描述}
- 生气/不开心信号：{描述}
- 开心信号：{描述}
- 试探模式：{描述或"无明显试探"}

### 关系动态
- 主动方：{ta/你/大致持平}
- 多想触发器：{列表}
- 满足感来源：{列表}
- 地雷话题：{列表}

### 兴趣度判断
- 阶段校准：{代号/材料名是否会误导；证据支持的真实阶段}
- 聊天甜度：{0-10 或 [待观察]}（证据：{一句}）
- 主动性：{0-10 或 [待观察]}（证据：{一句}）
- 关系承诺：{0-10 或 [待观察]}（证据：{一句}）
- 见面/行动兑现：{0-10 或 [待观察]}（证据：{一句}）
- 总体兴趣分：{0-10 或 [待观察]}（置信度：{高/中/低}）
- 加分信号：{列表}
- 减分信号：{列表}
- 建议投入：{继续/观察/降温/止损}

### 追法类型
- 类型：{类型名 + 置信度}
- 性别脚本：{追男生/追女生/不按性别套}
- 建议拉扯强度：{0/1/2/3}
- 有效动作：{展示自己/轻撩/降频观察/轻推回球/邀约测试}
- 加码窗口：{什么反馈出现时推进}
- 暂停主动窗口：{什么反馈出现时降频}

### 近期玩家信号

- 证据：{列表}
- 建议测试：{专属细节测试/低成本邀约测试/换时段测试/私聊兑现测试/节日兑现测试/降频观察}

### 朋友圈/社交动态画像
- 展示主题：{列表}
- 可能类型：{列表 + 证据}
- 外貌/妆容/穿搭：{展示风格 + 证据}
- 互动方式：{评论/点赞/回复语气}
- 可聊话题：{列表}
- 不建议碰：{列表}
- 用户展示自己方向：{列表}

### 微信息
- MBTI/星座态度：{信息 + 来源}
- 口头禅：{词 + 心情/场景}
- 心情变化话术：{开心/累/emo/生气时}
- 礼物/花/仪式感：{偏好/雷区}
- 饮食/作息/忌口：{信息}
- 前后不一致记录：{次数 + 证据，不直接定性}
- 表情包/梗偏好：{抽象/可爱/帅气/长辈图反差/不用等 + 证据}

### 潜台词字典
（ta专属的说话习惯解码）
| ta说的 | 可能需要的 |
|--------|---------|
| {具体说法} | {解读} |

### 性格底色与边界
（需要深判断、约会设计或人设选择时输出，详见 `references/personality_baseline.md`）
- 底色轴：{守规则↔挑战感等关键轴的偏向 + 证据}
- 气质偏好：{偏痞/偏乖/分情境}（用于 `references/persona_modes.md`）
- 喜欢/不喜欢：{各一句具体清单}
```
