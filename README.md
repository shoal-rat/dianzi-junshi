你和他/她之间，有时候差的不是感情，是那一句回复。

reply-skill 是一个运行在 Claude Code 中的恋爱回复助手。  
它帮你分析对方消息背后的真正意思，生成最能满足情感价值的回复方案——  
听起来像你说的话，不是 AI 写的稿。

---

## 核心功能

### 三层消息解析

每一条消息，都有三层含义：

```
第一层 · 表面意思   他说了什么
第二层 · 情绪状态   他现在感受如何
第三层 · 真正需要   他真正想要的是什么
```

"随便吃什么都行"背后可能是"我想让你帮我做决定"。  
"没事你忙吧"背后可能是"我很在意但我不想先开口"。  
reply-skill 帮你看穿这些。

### 情感价值驱动的回复

回复的目的不只是"回应"，而是让对方感受到情感价值：

| 情感价值 | 说明 |
|----------|------|
| 被珍视 | 他感受到你重视这段关系 |
| 被逗乐 | 他真的笑出来了 |
| 被理解 | 他觉得你懂他 |
| 被撩到 | 他感受到了你的心动 |
| 被安心 | 他不再焦虑或不安 |
| 被惊喜 | 他没想到你会这样说 |
| 被崇拜 | 他觉得你很有魅力 |

每个回复方案都会标注它主要传递的情感价值，和预期会引发的反应。

### 对象档案系统

你的他/她有独特的说话方式、情绪模式和潜台词习惯。  
reply-skill 为每个对象建立专属档案，越用越准：

- 说话风格分析（消息密度、标点习惯、常用语气词）
- 情感需求地图（爱的语言、被看见的需求、焦虑触发点）
- 行为模式识别（开心/难过/撒娇/生气时各自的表现）
- **潜台词词典**（记录他/她特有的说法和真实含义）

### 用户风格校准

生成的回复会对齐你的说话习惯——  
话少的人不会被建议说长篇大论，喜欢撩的人才会看到撩的方案。

## 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/shoal-rat/reply-skill.git
cd reply-skill

# 将 SKILL.md 注册到 Claude Code
# 在 Claude Code 中运行：
```

```
/reply-love
```

### 第一次使用

运行 `/reply-love`，系统会引导你完成 5 个问题：

```
Q1: 你的他/她叫什么？（起个代号就好）
Q2: 你们的关系基本情况
Q3: 他/她的性格特征
Q4: 你自己平时怎么回消息的？
Q5: 有没有聊天记录可以导入？（可选）
```

5 分钟建立档案，之后每次回复都更准。

## 使用示例

### 示例一：她说"你最近忙什么呢"

```
/reply 你最近忙什么呢
```

**消息解读**

```
表面：询问你最近的状态
情绪：轻微焦虑/在意 · 强度 2/5
真正需要：被重视——她想知道自己在你的生活里有没有分量
```

**回复方案**

```
【方案 1】暖心 · 情感价值：被珍视
————
"忙是忙，但没忙到忘了想你"
————
预期反应：她会觉得你心里有她，可能发个傻笑的表情
契合度：9/10

【方案 2】撩拨 · 情感价值：被撩到
————
"忙着想你啊，有什么问题吗"
————
预期反应：她可能翻白眼但会笑，然后继续聊
契合度：8/10

【方案 3】真诚 · 情感价值：被理解
————
"最近项目压力有点大，但跟你聊聊感觉会好一点，你最近怎么样"
————
预期反应：她会觉得被需要，话匣子打开
契合度：7/10
```

---

### 示例二：他说"没事，我不累"（明显是累的语气）

```
/reply 没事，我不累（他说话语气很沉，之前说要去加班了）
```

**消息解读**

```
表面：说自己不累
情绪：疲惫 + 轻微委屈 · 强度 3/5
真正需要：被心疼——他嘴上说不累，但想被看穿
```

潜台词提示：档案中记录 "没事" = 有事但不想主动说

**回复方案**

```
【方案 1】暖心 · 情感价值：被心疼
————
"我不信，加班完了记得告诉我，我等你"
————
预期反应：他会觉得被看穿了，可能语气软下来
契合度：9/10
```

---

### 示例三：她突然发来"你觉得我们合适吗"

```
/analyze 你觉得我们合适吗
```

**消息解读**

```
【表面意思】
询问你对这段关系是否合适的看法

【情绪状态】
焦虑/不安，强度：4/5
可能正在经历关系不确定感

【真正需要】
被安心——她不是真的在做数学题，她需要你告诉她"我们没问题"

【消息类型】测试型 · 【风险等级】需要关注

⚠️ 潜台词提示：这类问题通常是情绪出口，不是理性讨论，
   建议先安抚情绪，再表明立场。
```

## 聊天记录导入

支持导入聊天记录，让分析更准确：

```bash
# 处理 WeChatMsg 导出的 txt 文件
python tools/wechat_parser.py \
  --file chat_log.txt \
  --target "她的名字" \
  --output partners/xiaomei/materials/analysis.txt
```

支持格式：
- WeChatMsg 导出的 `.txt` 文件
- 手动粘贴的聊天记录（格式：`名字: 消息内容`）

## 反馈系统

发完消息后，告诉系统效果：

```
"我发了方案1，她回复很热情"   → 系统记录策略有效
"发了之后他就没回"             → 系统记录，下次调整
"这个建议不对，他不会这样"     → 立即更新档案
```

越用越懂你的对象。

## 注意事项

- **隐私**：所有档案数据存储在本地，不会发送到任何服务器
- **真实**：这个工具帮你找到合适的表达方式，不能替代真实的感情
- **边界**：不生成操控性、PUA 性质的回复建议
- **判断**：所有方案只是建议，你最了解你们的关系
